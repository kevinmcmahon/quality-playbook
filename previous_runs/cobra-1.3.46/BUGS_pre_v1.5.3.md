### BUG-001
- **Source:** Code Review
- **File:Line:** `command.go:827-857`
- **Severity:** High
- **Description:** `TraverseChildren` leaves explicit boolean literals in the command path, so `--verbose false child` does not route to `child`.
- **Minimal reproduction:** construct a root command with `TraverseChildren=true`, a parent-local boolean flag, and a `child` subcommand; execute `--verbose false child`.
- **Expected vs actual:** expected the child command to execute with `verbose=false`; actual behavior stays on the root command and prints root help because `false` is treated as a positional token.
- **Regression test:** `quality/test_regression_test.go::TestBug001TraverseChildrenTreatsExplicitFalseAsFlagValue`
- **Regression patch:** `quality/patches/BUG-001-regression-test.patch`
- **Fix patch:** none ÔÇö proposed diff included inline in `quality/writeups/BUG-001.md`
- **Spec basis:** `site/content/user_guide.md:284-295` documents `TraverseChildren` as ÔÇťparse local flags on each parent before executing the target command.ÔÇŁ

### BUG-002
- **Source:** Code Review
- **File:Line:** `completions.go:429-430`, `flag_groups.go:247-285`
- **Severity:** Medium
- **Description:** a completion request mutates live `Flag` objects, hiding mutually-exclusive siblings from later help output in the same process.
- **Minimal reproduction:** define mutually-exclusive `--json` / `--yaml` flags, request completion for `--json`, then call `--help` on the same `Command` tree.
- **Expected vs actual:** expected help to still show both `--json` and `--yaml`; actual help only shows `--json` because `yaml.Hidden` was set during completion.
- **Regression test:** `quality/test_regression_test.go::TestBug002CompletionDoesNotMutateHelpVisibility`
- **Regression patch:** `quality/patches/BUG-002-regression-test.patch`
- **Fix patch:** none ÔÇö proposed diff included inline in `quality/writeups/BUG-002.md`
- **Spec basis:** CobraÔÇÖs completion docs treat completion as a read-only query surface (`docs_gathered/06-SHELL_COMPLETIONS.md:24-35`, `181-227`); help rendering should continue to reflect the full flag surface.

### BUG-003
- **Source:** Spec Audit
- **File:Line:** `command.go:1007-1012`, `1263-1308`; `completions.go:230-306`
- **Severity:** Medium
- **Description:** required root persistent flags block builtin `help [command]` and `completion <shell>` helper commands.
- **Minimal reproduction:** mark root `--config` as required, then run `help child` or `completion bash` without `--config`.
- **Expected vs actual:** expected helper commands to stay accessible for discovery and shell setup; actual behavior errors with `required flag(s) "config" not set`.
- **Regression test:** `quality/test_regression_test.go::TestBug003BuiltinHelpersIgnoreRequiredPersistentRootFlags`
- **Regression patch:** `quality/patches/BUG-003-regression-test.patch`
- **Fix patch:** none ÔÇö proposed diff included inline in `quality/writeups/BUG-003.md`
- **Spec basis:** `docs_gathered/17-GITHUB_ISSUES_SUMMARY.md:44-50` and `docs_gathered/04-FLAG_INHERITANCE_AND_SCOPING.md:148-164` treat helper-command breakage behind required root flags as a limitation/workaround, not as intended behavior.

### BUG-004
- **Source:** Code Review (gap iteration)
- **File:Line:** `powershell_completions.go:98-105`, `187-195`
- **Severity:** Medium
- **Description:** PowerShell completion only recognizes `--flag=value` as an equal-sign flag request, so shorthand `-f=value` completion crashes instead of returning registered values.
- **Minimal reproduction:** define a flag with long + shorthand names, register a completion function, generate PowerShell completion, and request completion for `tool -f=`.
- **Expected vs actual:** expected the same candidates exposed for `--format=` (for example `-f=json` / `-f=yaml`); actual behavior leaves `-f=` in `$WordToComplete`, filters away all candidates, and PowerShell errors because Cobra emits a null completion text.
- **Regression test:** `quality/test_regression_test.go::TestBug004PowerShellShortFlagEqualsCompletionUsesRegisteredValues`
- **Regression patch:** `quality/patches/BUG-004-regression-test.patch`
- **Fix patch:** none ÔÇö proposed diff included inline in `quality/writeups/BUG-004.md`
- **Spec basis:** `docs_gathered/06-SHELL_COMPLETIONS.md:300-308` promises aligned cross-shell behavior for the shared Go completion logic, and `docs_gathered/14-PFLAG_LIBRARY.md:67-68` documents shorthand `-f=value` syntax as a supported Cobra/pflag form.

### BUG-005
- **Source:** Code Review (gap iteration)
- **File:Line:** `powershell_completions.go:179-185`
- **Severity:** Medium
- **Description:** PowerShell completion ignores `ShellCompDirectiveFilterFileExt` and falls back to generic path completion, so `MarkFlagFilename` suggestions include disallowed files.
- **Minimal reproduction:** mark a `--config` flag with `MarkFlagFilename("config", "yaml", "json")`, create `a.yaml`, `b.json`, and `c.txt`, generate PowerShell completion, and request `tool --config `.
- **Expected vs actual:** expected only `.yaml` / `.json` suggestions; actual completion includes `c.txt` because the PowerShell wrapper returns early and lets unfiltered filesystem completion take over.
- **Regression test:** `quality/test_regression_test.go::TestBug005PowerShellFilenameFilteringHonorsExtensions`
- **Regression patch:** `quality/patches/BUG-005-regression-test.patch`
- **Fix patch:** none ÔÇö proposed diff included inline in `quality/writeups/BUG-005.md`
- **Spec basis:** `docs_gathered/06-SHELL_COMPLETIONS.md:221-222` defines file/directory filter directives, and `docs_gathered/06-SHELL_COMPLETIONS.md:261-273` documents `MarkFlagFilename` / `MarkFlagDirname` as portable helpers rather than bash/zsh-only affordances.
