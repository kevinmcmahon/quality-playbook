## Domain and Stack

[Iteration 1] The repository is a specification-first quality skill with Python benchmark / validation helpers, generated `quality/` artifacts, and reference documents used by AI coding tools.

[Iteration 2: gap] The repo also ships user-facing orchestrator prompts and a local `pytest` shim that shape how maintainers invoke the skill.

[Iteration 3: unfiltered] The same product contract includes the runner’s warning semantics, generated child prompts, and the self-audit validator’s ability to classify the bootstrap repo correctly when fixture repos are nested inside the checkout.

[Iteration 4: parity] The contract also includes the quality gate's ability to discover and validate language-appropriate functional-test filenames consistently across the helper library, the file-existence gate, and the extension checker.

[Iteration 5: adversarial] The contract also includes the helper library's responsibility to honor only the published artifact names when reporting functional / regression coverage, rather than inventing extra aliases.

## Quality Risks

[Iteration 1] Baseline risks emphasized shallow runner coverage, benchmark bookkeeping drift, and quality-gate contract mismatches.

[Iteration 2: gap] Added risks:

1. README bootstrap failure for source-checkout users if orchestrators cannot find root `SKILL.md`
2. phase-boundary collapse when the general orchestrator both forbids and performs in-session execution
3. accidental test execution when a maintainer expects collection-only behavior
4. TDD / debug command crashes when a maintainer uses documented pytest node IDs

[Iteration 3: unfiltered] Added risks:

5. code-only audits become impossible because the docs-warning gate skips instead of warns
6. generated prompts fail on valid install layouts, producing child-model confusion before any phase logic runs
7. failed child runners can be reported as successful local runs, poisoning automation and iteration control flow
8. the self-audit validator can fail on the repo’s own supported layout because fixture repos redefine the detected language

[Iteration 4: parity] Added risks:

9. a valid Go repo can FAIL the file-existence gate solely because it uses the documented `functional_test.go` name
10. Java/Scala/TypeScript repos can bypass extension validation because the checker only recognizes `test_functional.*`

[Iteration 5: adversarial] Added risks:

11. the runner can suppress a missing-functional-test warning when the only artifact is the undocumented alias `test_functional_test.*`
12. benchmark summaries can overstate regression readiness because helper discovery accepts aliases the published contract does not
