## Demoted Candidates

### DC-001: Agent SKILL.md search order disagreement
- **Source:** [Iteration 2: gap] Finding 3
- **Dismissal reason:** The two agent files search paths in different order, but both find the correct SKILL.md as long as only one installation exists. The inconsistency only manifests if both flat and nested Copilot installations exist simultaneously, which is unusual.
- **Code location:** agents/quality-playbook.agent.md:17-19 vs agents/quality-playbook-claude.agent.md:21-23
- **Re-promotion criteria:** Show a scenario where both `.github/skills/SKILL.md` and `.github/skills/quality-playbook/SKILL.md` exist with different versions and the two agents produce different results.
- **Status:** FALSE POSITIVE [adversarial] — Agent files have no spec-defined contract for search order consistency. The inconsistency exists but is not a bug against any normative requirement.

### DC-002: Duplicate tdd-results.json version check
- **Source:** [Iteration 2: gap] Finding 7
- **Dismissal reason:** The duplicate check inflates the FAIL counter but does not cause a false pass (any non-zero FAIL count triggers exit 1). It's a code quality issue, not a spec violation.
- **Code location:** quality_gate.sh:594-598, 617-624
- **Re-promotion criteria:** Demonstrate that the duplicate FAIL causes a user to misinterpret the gate output (e.g., thinking there are two separate problems when there is only one).
- **Status:** DEMOTED

### DC-003: --all with empty VERSION
- **Source:** [Iteration 2: gap] Finding 8
- **Dismissal reason:** The script exits 1 with a usage message, which is a reasonable fallback. The error message could be more helpful, but the behavior is not incorrect.
- **Code location:** quality_gate.sh:60-67, 630-633
- **Re-promotion criteria:** Show that the user gets a confusing error message that leads them to a wrong fix (e.g., they think they need repo arguments instead of realizing VERSION is empty).
- **Status:** FALSE POSITIVE [adversarial] — "Version: unknown" in the header provides sufficient diagnostic context. Suboptimal UX but not a behavioral defect.

### DC-004: SKILL.md "nine files" count inconsistency
- **Source:** [Iteration 3: unfiltered] Finding 4
- **Dismissal reason:** The "nine files" count in the introductory table is a narrative summary of the primary deliverables. The "seven files below" in Phase 2 refers to the numbered File N sections. The disagreement is confusing documentation but does not cause incorrect behavior.
- **Code location:** SKILL.md:63, 906, 939-1323
- **Re-promotion criteria:** Show a scenario where an agent produces fewer artifacts because it counts "nine" from the table and stops before generating all artifacts that Phase 2 actually requires.
- **Status:** FALSE POSITIVE [adversarial] — The counts refer to different subsets (9 primary deliverables vs 7 Phase 2 numbered sections) and are both accurate in their contexts. Phase 2 explicitly states which files were already produced by the pipeline.

### DC-005: quality_gate.sh date validation accepts impossible dates
- **Source:** [Iteration 3: unfiltered] Finding 5
- **Dismissal reason:** The date field is generated by the agent, not user-supplied. AI agents always generate valid ISO 8601 dates. An impossible date like "2025-99-99" would only appear if the agent fabricated it, which is not a realistic failure mode.
- **Code location:** quality_gate.sh:257-273
- **Re-promotion criteria:** Show a real scenario where an agent generates an impossible date that passes the gate and causes downstream problems.
- **Status:** DEMOTED

### DC-006: SKILL.md Phase 7 has no completion gate
- **Source:** [Iteration 3: unfiltered] Finding 7
- **Dismissal reason:** Phase 7 is explicitly interactive with no defined endpoint (SKILL.md line 2027: "there's no explicit 'done' step"). The absence of a gate is intentional design.
- **Code location:** SKILL.md:1904-2028
- **Re-promotion criteria:** Show that the absence of a Phase 7 entry gate causes an agent to produce incorrect output when running Phase 7 before Phase 6 completes.
- **Status:** FALSE POSITIVE [adversarial] — Phase 7 is interactive by design. The lack of a mechanical entry gate is consistent with its interactive nature. REQ-003 already tracks the completion-gate gap.
