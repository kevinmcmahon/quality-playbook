### BUG-008: quality_gate.sh crashes on macOS bash 3.2 with no arguments [gap]

**Source:** Gap Iteration — Exploration Finding 1
**File:** quality_gate.sh:32, 639
**Severity:** MEDIUM
**Description:** The script sets `set -uo pipefail` (line 32) and initializes `REPO_DIRS=()` (line 37). When invoked with no arguments, the else branch at line 637 iterates `for name in "${REPO_DIRS[@]}"` over the empty array. On bash 3.2 (macOS default), `"${arr[@]}"` on an empty array with `set -u` triggers "unbound variable" instead of iterating zero times. The script crashes before reaching the usage message at line 654.

**Reproduction:** Run `./quality_gate.sh` on macOS (bash 3.2) with no arguments. Alternatively: `bash -c 'set -uo pipefail; arr=(); for x in "${arr[@]}"; do echo "$x"; done'`

**Expected:** Script prints usage message and exits 1.
**Actual:** Script crashes with "bash: REPO_DIRS[@]: unbound variable" and exits 127.

**Spec basis:** quality_gate.sh lines 25-27 define exit codes: 0 for pass, 1 for failure. Exit 127 is not a defined exit code. The usage message at line 654 is the intended behavior for no-arguments invocation.
