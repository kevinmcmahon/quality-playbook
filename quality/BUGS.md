### BUG-011: Citation verifier allows path traversal outside the repo root
- Primary requirement: REQ-011
- Severity: HIGH
- Source: Adversarial iteration Code Review
- File:line: `bin/citation_verifier.py:244-266`
- Spec basis: REQ-011 requires `verify_citation()` to reject `source_path` values that resolve outside `Path(root).resolve()` before reading bytes, so a tampered manifest or auditor-supplied citation cannot be turned into an out-of-tree file-read primitive (`bin/citation_verifier.py:244-266`, `quality/REQUIREMENTS.md:328-346`).
- Minimal reproduction:
  1. Construct a citation dict with `{"document": "../../etc/passwd"}` (or any absolute path).
  2. Call `verify_citation(citation, formal_doc, root=<repo_dir>)`.
  3. Observe that `read_bytes()` is invoked against the out-of-root resolved path.
- Expected behavior: `verify_citation()` returns a non-OK `VerificationResult` with a structured boundary error code (e.g. `ERROR_DOCUMENT_OUT_OF_ROOT`) before any file is read.
- Actual behavior: the verifier computes `doc_path = Path(root) / source_path` and immediately calls `doc_path.read_bytes()`. `Path(root) / "/etc/passwd"` evaluates to `/etc/passwd` (Python `Path.__truediv__` discards the left side when the right operand is absolute), and `Path(root) / "../../etc/passwd"` resolves outside `root`. No `is_relative_to(root.resolve())` check is performed.
- Regression test: `quality/test_regression.py::CodeReviewRegressionTests.test_bug_011_citation_verifier_rejects_out_of_root_paths`
- Regression patch: `quality/patches/BUG-011-regression-test.patch`
