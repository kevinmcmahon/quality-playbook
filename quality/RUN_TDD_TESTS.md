# TDD Verification Protocol: quality-playbook (QFB-bootstrap)

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## Overview

This protocol runs the red-green TDD cycle for each confirmed bug: the regression test fails on unpatched code (red), then passes after applying the fix patch (green). For this specification-primary Bash project, regression tests are shell scripts that verify quality_gate.sh behavior or SKILL.md/reference file content.

## Prerequisites

- `bash` available (version 3.2+)
- `quality/BUGS.md` populated with confirmed bugs
- `quality/patches/` directory with regression-test and fix patches
- `quality/test_regression.sh` with executable regression tests

## Spec-Grounded Test Requirements

For each bug in `quality/BUGS.md`:

1. **Read the bug's spec basis** to identify the documentation passage defining expected behavior
2. **Read the spec passage** (SKILL.md line range, reference file section) directly
3. **Write test assertions using language from the spec** — function names, artifact names, and severity levels should echo SKILL.md's terminology
4. **Include a comment block** in each test citing: requirement ID (from REQUIREMENTS.md), bug ID (from BUGS.md), and spec passage

## Red-Green Execution Steps

For each confirmed bug with a fix patch:

### Red Phase (mandatory for ALL confirmed bugs)

```bash
# Remove fix if previously applied
git apply -R quality/patches/BUG-NNN-fix.patch 2>/dev/null

# Run the specific regression test
TEST_CMD="bash quality/test_regression.sh test_bug_nnn"
OUTPUT=$($TEST_CMD 2>&1)
EXIT=$?

# Log result
printf 'RED\n--- Test output for BUG-NNN red phase ---\nCommand: %s\nExit code: %d\n%s\n' \
  "$TEST_CMD" "$EXIT" "$OUTPUT" > quality/results/BUG-NNN.red.log
```

**Expected:** Test fails (exit code 1). The red phase MUST run for every confirmed bug, even when no fix patch exists.

### Green Phase (when fix patch exists)

```bash
# Apply fix
git apply quality/patches/BUG-NNN-fix.patch

# Run the same test
OUTPUT=$($TEST_CMD 2>&1)
EXIT=$?

# Log result
printf 'GREEN\n--- Test output for BUG-NNN green phase ---\nCommand: %s\nExit code: %d\n%s\n' \
  "$TEST_CMD" "$EXIT" "$OUTPUT" > quality/results/BUG-NNN.green.log
```

**Expected:** Test passes (exit code 0).

### Record

Update PROGRESS.md BUG tracker with closure status:
- `TDD verified (FAIL->PASS)` — red failed, green passed
- `confirmed open (xfail)` — red failed, no fix patch
- `red failed` — test passed on unpatched code (test doesn't detect the bug)
- `green failed` — test still fails after fix (fix is incomplete)

## Framework Adaptation

This project uses Bash. Regression tests are shell functions in `quality/test_regression.sh`. Each test function:

1. Sets up a test directory with specific artifact configurations
2. Runs quality_gate.sh or greps source files
3. Asserts expected behavior (exit code, output content)
4. Returns 0 for pass, 1 for fail

**Test function naming convention:** `test_bug_NNN_short_description`

**Expected-failure mechanism:** Since Bash has no built-in xfail, regression tests that target unfixed bugs should be documented in the test file header with a comment: `# XFAIL: BUG-NNN — unskip after applying quality/patches/BUG-NNN-fix.patch`

## Upstream Reporting Format

For each TDD-verified bug, generate a ready-to-send report block:

```
## BUG-NNN: [one-sentence description citing spec section]

### FAIL->PASS Output
```
$ bash quality/test_regression.sh test_bug_nnn  # BEFORE fix
FAIL: [description]
Exit code: 1

$ git apply quality/patches/BUG-NNN-fix.patch
$ bash quality/test_regression.sh test_bug_nnn  # AFTER fix
PASS: [description]
Exit code: 0
```

### Test File
[quality/test_regression.sh — test_bug_nnn function]

### Fix Patch
[quality/patches/BUG-NNN-fix.patch]
```
