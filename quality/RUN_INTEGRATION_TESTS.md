# Integration Test Protocol: quality-playbook (QFB-bootstrap)

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## Working Directory

All commands in this protocol use **relative paths from the project root.** Run everything from the QFB-bootstrap directory. Do not `cd` to an absolute path.

## Safety Constraints

- DO NOT modify SKILL.md, quality_gate.sh, or any reference file
- ONLY create files in the test results directory (quality/results/)
- If something fails, record it and move on — DO NOT fix it

## Pre-Flight Check

Before running integration tests, verify:
- [ ] `quality_gate.sh` is executable: `chmod +x quality_gate.sh && ls -la quality_gate.sh`
- [ ] bash is available: `bash --version`
- [ ] SKILL.md exists in repo root: `test -f SKILL.md && echo OK`
- [ ] quality/ directory exists with artifacts: `ls quality/*.md | wc -l`

## Automated Integration Tests

### Group 1: Gate on Complete Artifacts (UC-02)

```bash
# Create a complete test directory
TEST_DIR=$(mktemp -d)
mkdir -p "${TEST_DIR}/quality"
for f in BUGS.md REQUIREMENTS.md QUALITY.md PROGRESS.md COVERAGE_MATRIX.md COMPLETENESS_REPORT.md CONTRACTS.md RUN_CODE_REVIEW.md RUN_SPEC_AUDIT.md RUN_INTEGRATION_TESTS.md RUN_TDD_TESTS.md; do
    echo "# ${f}" > "${TEST_DIR}/quality/${f}"
done
echo "#!/bin/bash" > "${TEST_DIR}/quality/test_functional.sh"
cp SKILL.md "${TEST_DIR}/"

bash quality_gate.sh "${TEST_DIR}" 2>&1 | tee quality/results/integration-group-1.log
echo "Exit: $?" >> quality/results/integration-group-1.log

rm -rf "${TEST_DIR}"
```

**Pass criteria:** Exit 0. No "FAIL:" lines in output.

### Group 5: Strictness Modes (UC-02, UC-07)

```bash
TEST_DIR=$(mktemp -d)
mkdir -p "${TEST_DIR}/quality"
for f in BUGS.md REQUIREMENTS.md QUALITY.md PROGRESS.md COVERAGE_MATRIX.md COMPLETENESS_REPORT.md; do
    echo "# ${f}" > "${TEST_DIR}/quality/${f}"
done
cp SKILL.md "${TEST_DIR}/"

echo "--- Benchmark mode ---" > quality/results/integration-group-5.log
bash quality_gate.sh --benchmark "${TEST_DIR}" >> quality/results/integration-group-5.log 2>&1
echo "Benchmark exit: $?" >> quality/results/integration-group-5.log

echo "--- General mode ---" >> quality/results/integration-group-5.log
bash quality_gate.sh --general "${TEST_DIR}" >> quality/results/integration-group-5.log 2>&1
echo "General exit: $?" >> quality/results/integration-group-5.log

rm -rf "${TEST_DIR}"
```

**Pass criteria:** Both modes run without errors. Output shows different WARN/FAIL behavior for strictness-sensitive checks.

### Group 6: Multiple Repo Validation (UC-07)

```bash
DIR1=$(mktemp -d)
DIR2=$(mktemp -d)
for d in "$DIR1" "$DIR2"; do
    mkdir -p "${d}/quality"
    for f in BUGS.md REQUIREMENTS.md QUALITY.md PROGRESS.md COVERAGE_MATRIX.md COMPLETENESS_REPORT.md; do
        echo "# ${f}" > "${d}/quality/${f}"
    done
done
cp SKILL.md "$DIR1/" "$DIR2/"

bash quality_gate.sh "$DIR1" "$DIR2" 2>&1 | tee quality/results/integration-group-6.log
echo "Exit: $?" >> quality/results/integration-group-6.log

rm -rf "$DIR1" "$DIR2"
```

**Pass criteria:** Output shows two separate repo headings ("=== repo1 ===", "=== repo2 ==="). Each repo evaluated independently.

### Group 7: SKILL.md Internal Consistency (UC-03)

```bash
echo "--- Artifact Contract Consistency Check ---" > quality/results/integration-group-7.log

# Extract Required: Yes artifacts from SKILL.md
echo "SKILL.md Required: Yes artifacts:" >> quality/results/integration-group-7.log
grep "| Yes |" SKILL.md | sed 's/|/\t/g' | awk '{print $2}' >> quality/results/integration-group-7.log

# Extract checked artifacts from quality_gate.sh
echo "" >> quality/results/integration-group-7.log
echo "quality_gate.sh checked artifacts:" >> quality/results/integration-group-7.log
grep -E "^\s+for f in|test_functional" quality_gate.sh | head -5 >> quality/results/integration-group-7.log

echo "Manual comparison required" >> quality/results/integration-group-7.log
```

**Pass criteria:** Both lists exist for manual comparison. (Full mechanical comparison in test_functional.sh)

### Group 8: Reference File Availability (UC-04)

```bash
echo "--- Reference File Check ---" > quality/results/integration-group-8.log
MISSING=0
for f in references/exploration_patterns.md references/review_protocols.md references/requirements_pipeline.md references/spec_audit.md references/verification.md references/iteration.md references/constitution.md references/functional_tests.md references/defensive_patterns.md references/schema_mapping.md references/requirements_refinement.md references/requirements_review.md; do
    if [ -f "$f" ]; then
        echo "FOUND: $f ($(wc -l < "$f") lines)" >> quality/results/integration-group-8.log
    else
        echo "MISSING: $f" >> quality/results/integration-group-8.log
        MISSING=$((MISSING + 1))
    fi
done
echo "Missing: ${MISSING}" >> quality/results/integration-group-8.log
```

**Pass criteria:** All 12 reference files found. Missing count is 0.
