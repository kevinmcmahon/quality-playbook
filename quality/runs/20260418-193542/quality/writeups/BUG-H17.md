# BUG-H17 Writeup: BUG-[0-9]+ Regex Never Matches Severity-Prefix IDs — Gate Completely Bypassed

**Severity**: HIGH
**File:Line**: `quality_gate.sh:184` (bug count), `quality_gate.sh:313` (ID extraction)
**Requirement**: REQ-020

---

## Summary

`quality_gate.sh` counts BUGS.md bug headings using the regex `^### BUG-[0-9]+` (line 184) and extracts IDs with `BUG-[0-9]+` (line 313). Both regexes require a purely numeric suffix. The QFB generates severity-prefixed IDs (`BUG-H1`, `BUG-M3`, `BUG-L6`) where the suffix is a severity letter followed by a number. For any BUGS.md using severity-prefix IDs, `bug_count` is set to 0 and `bug_ids` is empty — causing ALL downstream validation (TDD logs, patches, writeups) to be silently skipped. The gate reports PASS while providing zero artifact validation.

---

## Root Cause

```diff
- correct_headings=$(grep -cE '^### BUG-[0-9]+' "${q}/BUGS.md" || true)
+ correct_headings=$(grep -cE '^### BUG-([HML]|[0-9])[0-9]*' "${q}/BUGS.md" || true)
```

```diff
- bug_ids=$(grep -oE 'BUG-[0-9]+' "${q}/BUGS.md" 2>/dev/null \
-     | grep -E '^BUG-[0-9]+$' | sort -u -t'-' -k2,2n)
+ bug_ids=$(grep -oE 'BUG-([HML][0-9]+|[0-9]+)' "${q}/BUGS.md" 2>/dev/null \
+     | grep -E '^BUG-([HML][0-9]+|[0-9]+)$' | sort -u)
```

Direct proof that the old regex fails:
```bash
echo '### BUG-H1' | grep -cE '^### BUG-[0-9]+'
# Output: 0  (bug_count=0, ALL validation skipped)

echo '### BUG-H1' | grep -cE '^### BUG-([HML]|[0-9])[0-9]*'
# Output: 1  (bug_count=1, validation proceeds correctly)
```

---

## Impact — Complete Gate Bypass

When `bug_count=0`:
- **Line 223**: `if [ "$bug_count" -gt 0 ]` → FALSE → `INFO: Zero bugs — tdd-results.json not required`
- **Line 309**: `if [ "$bug_count" -gt 0 ]` → FALSE → `INFO: Zero bugs — TDD log files not required`
- **Line 564**: `if [ "$bug_count" -gt 0 ]` → FALSE → patches not validated
- **Line 592**: `if [ "$bug_count" -gt 0 ]` → FALSE → writeups not validated

The gate outputs `PASS` while having performed zero artifact validation. An agent using severity-prefix IDs (as generated by QFB Phase 3) can submit BUGS.md with any content, missing TDD logs, absent patches, and missing writeups — all will pass the gate without any checks.

This affects every standard QFB self-audit run, as Phase 3 generates severity-prefix IDs (`BUG-H1`, `BUG-M3`, `BUG-L6`) by design. The entire quality assurance mechanism is non-functional for the QFB's own intended use.

---

## Fix

Extend the regex to match both formats — severity-prefix (`BUG-H1`) and numeric (`BUG-001`):

```diff
- correct_headings=$(grep -cE '^### BUG-[0-9]+' "${q}/BUGS.md" || true)
+ correct_headings=$(grep -cE '^### BUG-([HML]|[0-9])[0-9]*' "${q}/BUGS.md" || true)
```

```diff
- bug_ids=$(grep -oE 'BUG-[0-9]+' "${q}/BUGS.md" 2>/dev/null \
-     | grep -E '^BUG-[0-9]+$' | sort -u -t'-' -k2,2n)
+ bug_ids=$(grep -oE 'BUG-([HML][0-9]+|[0-9]+)' "${q}/BUGS.md" 2>/dev/null \
+     | grep -E '^BUG-([HML][0-9]+|[0-9]+)$' | sort -u)
```

Also update the format-policing patterns at lines 186, 189, 191, 193 to use `BUG-([HML]|[0-9])` rather than `BUG-[0-9]+`. Additionally, SKILL.md:1615 should be updated from `"e.g., BUG-001"` to `"e.g., BUG-001 or BUG-H1"` to document both valid formats.
