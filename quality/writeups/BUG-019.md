# BUG-019: verification.md benchmark 40 omits 7 required artifacts from file-existence check

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## 1. Summary

verification.md benchmark 40 defines the artifact file-existence gate for Phase 6, listing the files that must exist before Phase 5 is marked complete. The benchmark lists 6 unconditionally-required files: BUGS.md, REQUIREMENTS.md, QUALITY.md, PROGRESS.md, COVERAGE_MATRIX.md, COMPLETENESS_REPORT.md. SKILL.md's artifact contract table marks 13+ artifacts as "Required: Yes," including 7 that benchmark 40 omits entirely: CONTRACTS.md, test_functional.*, RUN_CODE_REVIEW.md, RUN_INTEGRATION_TESTS.md, RUN_SPEC_AUDIT.md, RUN_TDD_TESTS.md, and AGENTS.md.

## 2. Spec reference

SKILL.md line 85: "any gate check should trace to an artifact listed here." The artifact contract table defines the authoritative list. verification.md benchmark 40 is the corresponding verification checkpoint. The two should agree.

## 4. Observable consequence

An agent implementing Phase 6 verification by following benchmark 40's file list literally would check only 6 files. If all 7 omitted artifacts are missing, the agent would report benchmark 40 as PASSED. The run would appear to pass Phase 6 verification with half the required artifacts absent. Note: SKILL.md's own Phase 5 artifact gate (lines 1627-1641) lists a more complete set, so an agent that follows SKILL.md's instructions directly would catch the missing files. The bug is specifically in the benchmark document that Phase 6 references.

## 6. The fix

Add the missing artifacts to benchmark 40's required-files list:

```diff
--- a/references/verification.md
+++ b/references/verification.md
@@ -226,7 +226,7 @@
 ### 40. Artifact File-Existence Gate Passed

-Before Phase 5 is marked complete, verify that all required artifacts exist as files on disk — not just referenced in PROGRESS.md. Required files: BUGS.md, REQUIREMENTS.md, QUALITY.md, PROGRESS.md, COVERAGE_MATRIX.md, COMPLETENESS_REPORT.md.
+Before Phase 5 is marked complete, verify that all required artifacts exist as files on disk — not just referenced in PROGRESS.md. Required files: BUGS.md, REQUIREMENTS.md, QUALITY.md, PROGRESS.md, COVERAGE_MATRIX.md, COMPLETENESS_REPORT.md, CONTRACTS.md, test_functional.*, RUN_CODE_REVIEW.md, RUN_INTEGRATION_TESTS.md, RUN_SPEC_AUDIT.md, RUN_TDD_TESTS.md, AGENTS.md.
```
