# BUG-002: quality_gate.sh test_functional detection misses SKILL.md-allowed alternative names

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## 1. Summary

quality_gate.sh line 125 only checks `ls ${q}/test_functional.*` for functional test files. SKILL.md line 69 tells agents to use language-appropriate names like `FunctionalTest.java`, `FunctionalSpec.scala`, and `functional.test.ts`. Projects following SKILL.md guidance get a false WARN.

## 2. Spec reference

- SKILL.md line 69: "Use the project's language: `test_functional.py` (Python), `FunctionalSpec.scala` (Scala), `functional.test.ts` (TypeScript), `FunctionalTest.java` (Java), etc."
- Requirement: REQ-007

## 3. Code citation

quality_gate.sh line 125:
```bash
if ls ${q}/test_functional.* &>/dev/null; then
```

Only matches the `test_functional.*` pattern, missing `FunctionalTest.*`, `FunctionalSpec.*`, and `functional.test.*`.

## 4. Observable consequence

A Java project with `FunctionalTest.java` in quality/ gets "test_functional.* missing" WARN despite having a correctly-named functional test file per SKILL.md guidance.

## 5. Test description

Two regression tests:
- `test_bug_002_functional_test_java_name`: Creates FunctionalTest.java, removes test_functional.sh, checks gate output.
- `test_bug_002_functional_test_ts_name`: Creates functional.test.ts, removes test_functional.sh, checks gate output.

TDD result: RED -> GREEN.
