# BUG-022 — Child runner failures are reported as successful phases/runs

## 1. Summary

`run_prompt()` already returns child exit status, but its callers drop it. A failed Claude/Copilot
child can therefore be followed by `Phase complete` / `Playbook complete` logging and a zero exit
status.

## 2. Spec reference

REQ-023 — Child prompt failures must propagate to runner status.

> `[Tier 2] [source] bin/run_playbook.py:516-551` — "`run_prompt` returns process exit status"

## 3. The code

`bin/run_playbook.py:600-642` and `671-705` call `run_prompt(...)` but do not inspect its return
value before logging success. `execute_run()` then trusts the per-repo exit code.

## 4. Observable consequence

A run can fail in the child process but still look successful to the operator and to automation.
In the single-pass path this also suppresses failure counting upstream.
