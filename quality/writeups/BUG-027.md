# BUG-027 — Helper summary counts non-canonical regression aliases as coverage

## 1. Summary

The helper library's regression discovery accepts `RegressionTest.*`, `regression_test.*`, and
`test_regression_test.*`, then `build_summary_rows()` reports `REGR=Y` for any such match. The
published artifact contract, however, is canonical `quality/test_regression.*`. The summary can
therefore overstate regression readiness for a non-conformant repo.

## 2. Spec reference

REQ-028 — Helper regression discovery reports only canonical `test_regression.*`.

> `[Tier 1] [SKILL.md] §Complete Artifact Contract` — "Regression tests | `quality/test_regression.*`"

## 3. The code

`bin/benchmark_lib.py:28-33` defines the permissive `REGRESSION_TEST_PATTERNS`. `find_regression_test()`
at `bin/benchmark_lib.py:168-169` exposes that set, and `build_summary_rows()` at
`bin/benchmark_lib.py:284-308` turns any hit into `REGR=Y`. The helper summary therefore disagrees
with both the published contract and `quality_gate.py:894-901`, which still require canonical
`test_regression.*`.

## 4. Observable consequence

A benchmark operator comparing runs can believe a repo has regression coverage when it only has a
helper-only alias. That false positive undermines the artifact summary's purpose: quickly telling a
maintainer which runs are actually ready for closure.

## 6. The fix

```diff
--- a/bin/benchmark_lib.py
+++ b/bin/benchmark_lib.py
@@
 REGRESSION_TEST_PATTERNS = (
     "test_regression.*",
-    "regression_test.*",
-    "test_regression_test.*",
-    "RegressionTest.*",
 )
diff --git a/bin/tests/test_benchmark_lib.py b/bin/tests/test_benchmark_lib.py
--- a/bin/tests/test_benchmark_lib.py
+++ b/bin/tests/test_benchmark_lib.py
@@
-            regression = repo_dir / "quality" / "RegressionTest.java"
+            regression = repo_dir / "quality" / "test_regression.py"
             write(functional, "ok")
             write(regression, "ok")
```

## 7. The test

`quality/test_regression.py::AdversarialIterationRegressions::test_reg_adv27_summary_ignores_noncanonical_regression_aliases`
builds a minimal repo whose only regression-looking file is `quality/RegressionTest.java`. On
current code, `build_summary_rows()` reports `REGR=Y` and the test fails. After narrowing helper
discovery to canonical `test_regression.*`, the summary leaves `REGR` unset and the test passes.
