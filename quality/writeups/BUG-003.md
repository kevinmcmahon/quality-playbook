# BUG-003: quality_gate.sh eval injection with user-supplied directory paths

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## 1. Summary

Lines 439-448 of quality_gate.sh use `eval "find '${repo_dir}' ..."` to detect project language. The `repo_dir` variable comes from command-line arguments. A directory name containing a single quote breaks the quoting and allows shell command injection. The eval is unnecessary since find can be called directly.

## 2. Spec reference

- Code quality finding (Tier 3): Shell scripts should not use eval with user-supplied arguments when direct execution achieves the same result.
- Requirement: REQ-009

## 3. Code citation

quality_gate.sh lines 438-448:
```bash
local find_exclude="-not -path '*/vendor/*' -not -path '*/node_modules/*' -not -path '*/.git/*' -not -path '*/quality/*'"
if eval "find '${repo_dir}' -maxdepth 3 ${find_exclude} -name '*.go' -print -quit" 2>/dev/null | grep -q .; then detected_lang="go"
```

The pattern repeats 10 times for different file extensions.

## 4. Observable consequence

A directory named `test'$(echo INJECTED)'dir` causes the injected command to execute when quality_gate.sh processes it. This is exploitable if quality_gate.sh is run on untrusted directory structures.

## 5. Test description

`test_bug_003_eval_usage_in_gate`: Source-inspection test that greps quality_gate.sh for `eval.*repo_dir` patterns. On unpatched code, finds 10 occurrences (XFAIL). After fix, finds 0 (PASS).

TDD result: RED -> GREEN.
