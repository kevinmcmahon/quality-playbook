# BUG-003: Hardcoded .github/skills/references/ Paths Break Claude Code Installations

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (bootstrap self-audit)

## 1. Summary

SKILL.md hardcodes `.github/skills/references/` paths in 16+ locations across phase instructions. When the skill is installed at `.claude/skills/quality-playbook/`, these paths do not resolve. Every Claude Code user encounters reference-not-found errors at the start of each phase.

## 2. Spec reference

docs_gathered/readme.md lines 26-30: Documents both `.github/skills/` (Copilot) and `.claude/skills/quality-playbook/` (Claude Code) installation paths.

The behavioral contract: reference file paths must resolve at all supported installation locations.

## 3. The code

```
SKILL.md:872
> - `.github/skills/references/requirements_pipeline.md` — ...

SKILL.md:1559
> - `.github/skills/references/requirements_pipeline.md` — post-review reconciliation process
```

16+ similar hardcoded paths throughout the document.

## 4. Observable consequence

A Claude Code user installs the skill at `.claude/skills/quality-playbook/SKILL.md`. When the agent reaches Phase 2 and tries to read `.github/skills/references/requirements_pipeline.md`, the file does not exist. The agent either fails or produces incomplete output. The orchestrator agents correctly check multiple SKILL.md locations, but SKILL.md itself does not adapt its internal paths.

## 6. The fix

```diff
--- a/SKILL.md
+++ b/SKILL.md
@@ -870,7 +870,7 @@
 > **Required references for this phase:**
-> - `.github/skills/references/requirements_pipeline.md` — five-phase requirements generation pipeline
+> - `references/requirements_pipeline.md` — five-phase requirements generation pipeline
```

Apply this pattern to all 16+ hardcoded occurrences. Change `.github/skills/references/X.md` to `references/X.md` (relative to SKILL.md location). Alternative: add path resolution logic at the top of SKILL.md.
