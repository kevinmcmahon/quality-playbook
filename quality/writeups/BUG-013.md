# BUG-013: quality_gate.sh resolved=() empty array crashes on bash 3.2

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## 1. Summary

The `resolved=()` array at quality_gate.sh line 638 triggers an "unbound variable" crash on bash 3.2 when all user-provided repo names fail to resolve. This is a second instance of the same bug class as BUG-008 (empty array + `set -u` on bash 3.2).

## 2. Spec reference

quality_gate.sh lines 25-27 define exit codes: 0 for pass, 1 for failure. Exit 127 from an "unbound variable" crash is not a defined exit code.

## 3. The code

quality_gate.sh:638 initializes `resolved=()`. Lines 639-648 loop over REPO_DIRS trying to resolve each name. If all names fail the `-d` checks, `resolved` stays empty. Line 650 does `REPO_DIRS=("${resolved[@]}")`, which on bash 3.2 with `set -u` (line 32) triggers "unbound variable."

## 4. Observable consequence

Running `./quality_gate.sh nonexistent_repo` on macOS (bash 3.2) crashes with "bash: resolved[@]: unbound variable" and exit 127, instead of printing "WARNING: Cannot find repo" followed by the usage message.

## 6. The fix

Use the bash 3.2-safe empty-array pattern:

```diff
--- a/quality_gate.sh
+++ b/quality_gate.sh
@@ -647,7 +647,7 @@
             echo "WARNING: Cannot find repo '${name}'"
         fi
     done
-    REPO_DIRS=("${resolved[@]}")
+    REPO_DIRS=("${resolved[@]+"${resolved[@]}"}")
 fi
```

## 7. The test

Run `bash -c 'set -uo pipefail; resolved=(); REPO_DIRS=("${resolved[@]}"); echo survived'`. On unpatched code, this crashes with "unbound variable" (exit 127). After the fix, it assigns an empty array and survives.
