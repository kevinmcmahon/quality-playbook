# BUG-010: Incomplete writeups get WARN instead of FAIL despite being spec-mandatory

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## 1. Summary

quality_gate.sh line 561 uses `warn()` when some but not all confirmed bugs have writeups. SKILL.md line 1454 says "Writeup generation for all confirmed bugs (mandatory)" and verification.md benchmark 30 says "Every confirmed bug... must have a writeup." The gate should `fail()` on incomplete writeups, not `warn()`, because the specification treats writeup completeness as a hard requirement.

## 2. Spec reference

SKILL.md line 1454: "Generate a writeup at `quality/writeups/BUG-NNN.md` for every confirmed bug -- both TDD-verified and confirmed-open." The word "every" combined with "mandatory" in the section heading establishes this as a hard requirement.

verification.md benchmark 30: "Every confirmed bug (TDD-verified or confirmed-open) must have a writeup at `quality/writeups/BUG-NNN.md`."

## 3. The code

```bash
# quality_gate.sh lines 558-564
if [ "$writeup_count" -ge "$bug_count" ]; then
    pass "${writeup_count} writeup(s) for ${bug_count} bug(s)"
elif [ "$writeup_count" -gt 0 ]; then
    warn "${writeup_count} writeup(s) for ${bug_count} bug(s) — incomplete"  # <-- should be fail
else
    fail "No writeups for ${bug_count} confirmed bug(s)"
fi
```

Note that the zero-writeup case (line 563) correctly uses `fail()`. Only the partial-writeup case (line 561) is downgraded to `warn()`. This creates an inconsistent severity profile: 0 writeups = FAIL, 1 writeup for 5 bugs = WARN. Both cases violate the same spec requirement.

## 4. Observable consequence

A run with 5 confirmed bugs and 3 writeups passes the quality gate (exit 0). The gate reports 0 FAILs and 1 WARN. The user sees "GATE PASSED" despite 2 bugs lacking mandatory writeups. This is the same class of issue as BUG-001 (required artifacts downgraded to WARN).

## 6. The fix

```diff
--- a/quality_gate.sh
+++ b/quality_gate.sh
@@ -558,7 +558,7 @@ if [ "$bug_count" -gt 0 ]; then
         if [ "$writeup_count" -ge "$bug_count" ]; then
             pass "${writeup_count} writeup(s) for ${bug_count} bug(s)"
         elif [ "$writeup_count" -gt 0 ]; then
-            warn "${writeup_count} writeup(s) for ${bug_count} bug(s) — incomplete"
+            fail "${writeup_count} writeup(s) for ${bug_count} bug(s) — incomplete (all confirmed bugs require writeups)"
         else
             fail "No writeups for ${bug_count} confirmed bug(s)"
         fi
```

## 7. The test

Create a quality directory with 3 confirmed bugs (### BUG-001 through ### BUG-003 in BUGS.md) and only 2 writeups (BUG-001.md and BUG-002.md). Run quality_gate.sh. Without fix: exit 0 (WARN only). With fix: exit 1 (FAIL).
