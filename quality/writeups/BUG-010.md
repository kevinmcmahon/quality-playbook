## Summary
The Claude-specific orchestrator teaches nested Copilot before flat Copilot, so the repo's two shipped orchestrator entry points no longer agree on the documented fallback order.

## Spec reference
- Requirement: REQ-010
- Spec basis: `quality/REQUIREMENTS.md:306-323`
- Behavioral contract quote: "shipped orchestrator entry points teach the same documented four-path fallback order"

## The code
`agents/quality-playbook.agent.md:37-45` matches the live skill's canonical order, but `agents/quality-playbook-claude.agent.md:47-54` lists `.github/skills/quality-playbook/SKILL.md` before `.github/skills/SKILL.md`. That reversal changes which valid Copilot install path the Claude orchestrator treats as the preferred first hit.

## Observable consequence
Two official orchestrator entry points now give different path-resolution instructions for the same repository. An operator following the Claude agent brief can resolve a different SKILL path than an operator following the generic orchestrator or the live skill.

## The fix
Rewrite the Claude orchestrator setup list so it matches the live skill and generic orchestrator exactly.
