# BUG-011: TDD_TRACEABILITY.md not checked by quality_gate.sh

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## 1. Summary

quality_gate.sh has no check for TDD_TRACEABILITY.md despite SKILL.md line 1426 and verification.md benchmark 28 both declaring it mandatory when confirmed bugs have red-phase results. A run that produces confirmed bugs with TDD verification but no TDD_TRACEABILITY.md will pass the gate.

## 2. Spec reference

SKILL.md line 1426: "If any bug has a red-phase result (whether TDD-verified or confirmed-open), `quality/TDD_TRACEABILITY.md` is also mandatory."

## 3. The code

quality_gate.sh lines 293-365 implement the [TDD Log Files] check section, which validates BUG-NNN.red.log and BUG-NNN.green.log files. There is no corresponding check for TDD_TRACEABILITY.md anywhere in the 677-line script.

## 4. Observable consequence

A playbook run that confirms bugs and runs TDD verification but skips generating TDD_TRACEABILITY.md (the spec-to-test mapping table) passes the gate with 0 FAILs. The traceability chain from spec passage to test function is broken without this file.

## 6. The fix

Add a TDD_TRACEABILITY.md check after the TDD log file checks section:

```diff
--- a/quality_gate.sh
+++ b/quality_gate.sh
@@ -365,6 +365,17 @@
         info "Zero bugs — TDD log files not required"
     fi
 
+    # --- TDD Traceability (benchmark 28) ---
+    echo "[TDD Traceability]"
+    if [ "$bug_count" -gt 0 ]; then
+        if [ "$red_found" -gt 0 ] 2>/dev/null; then
+            if [ -f "${q}/TDD_TRACEABILITY.md" ]; then
+                pass "TDD_TRACEABILITY.md exists"
+            else
+                fail "TDD_TRACEABILITY.md missing (mandatory when bugs have red-phase results)"
+            fi
+        fi
+    fi
+
     # --- Integration sidecar JSON — deep validation ---
```

## 7. The test

The regression test checks whether quality_gate.sh contains any mention of TDD_TRACEABILITY. On unpatched code, `grep -c TDD_TRACEABILITY quality_gate.sh` returns 0, confirming the check is absent.
