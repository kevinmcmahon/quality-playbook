# Bug Writeup: BUG-011
> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.5.3 — Andrew Stellman
> Date: 2026-04-28 · Project: quality-playbook

## Summary
`verify_citation()` resolves a caller-supplied `source_path` against the repo root with `Path(root) / source_path` and immediately calls `read_bytes()`, with no containment check. An absolute path is honored verbatim because `Path.__truediv__` discards the left side when the right operand is absolute, and a `..`-traversing relative path resolves outside `root` after `read_bytes()` runs. The verifier becomes an inadvertent file-read primitive when manifest or auditor input is hostile.

## Spec reference
- Requirement: REQ-011
- Spec basis: `quality/REQUIREMENTS.md:328-346`
- Behavioral contract quote: "verify_citation() must reject source_path values that resolve outside the supplied root (absolute paths, .. traversal, symlink escapes) before reading bytes from disk"

## The code
`bin/citation_verifier.py:244-266` reads:

```
source_path = citation.get("document") or formal_doc.get("source_path")
if not source_path:
    return VerificationResult(ok=False, error_code=ERROR_DOCUMENT_NOT_FOUND, ...)
doc_path = Path(root) / source_path
try:
    document_bytes = doc_path.read_bytes()
```

There is no `doc_path.resolve().is_relative_to(Path(root).resolve())` check before `read_bytes()`. `source_path` flows from two trust boundaries: `quality/formal_docs_manifest.json` (a gate-validated artifact, but the manifest writer can carry any `source_path` value) and the citation dict (produced from auditor responses in Phase 4). Either surface can carry an absolute path or a `..`-traversing relative path that resolves outside `root`.

## Observable consequence
A reduced reproduction confirms the read happens and returns a populated excerpt:

```
>>> from bin import citation_verifier
>>> from pathlib import Path
>>> import tempfile
>>> with tempfile.TemporaryDirectory() as outside, tempfile.TemporaryDirectory() as repo:
...     sentinel = Path(outside) / "secret.txt"
...     sentinel.write_text("alpha\nbeta\ngamma\n")
...     citation = {"document": str(sentinel), "line": 1}
...     formal_doc = {"source_path": str(sentinel)}
...     result = citation_verifier.verify_citation(citation, formal_doc, Path(repo))
...     print(result.ok, repr(result.excerpt))
True 'alpha'
```

The same shape is produced when `source_path` is a `..`-traversing relative path that resolves to the same out-of-root file. The gate's invariant is "the manifest is verified" — but the verifier silently treats any reachable file on the workstation as a valid citation source.

## The fix
Add a containment check in `verify_citation()` that resolves `doc_path` against `Path(root).resolve()` and returns a structured boundary error before any bytes are read. Reject absolute `source_path` values directly; reject relative paths whose resolved location is not under the resolved root; reject symlinks whose target escapes the root.

```diff
diff --git a/bin/citation_verifier.py b/bin/citation_verifier.py
--- a/bin/citation_verifier.py
+++ b/bin/citation_verifier.py
@@
     source_path = citation.get("document") or formal_doc.get("source_path")
     if not source_path:
         return VerificationResult(
             ok=False,
             error_code=ERROR_DOCUMENT_NOT_FOUND,
             error_message="citation has no 'document' field and formal_doc has no 'source_path'",
         )
 
-    doc_path = Path(root) / source_path
+    candidate = Path(source_path)
+    if candidate.is_absolute():
+        return VerificationResult(
+            ok=False,
+            error_code=ERROR_DOCUMENT_OUT_OF_ROOT,
+            error_message=f"source_path {source_path!r} is absolute; refusing to read outside repo root",
+        )
+    doc_path = (Path(root) / candidate).resolve()
+    try:
+        doc_path.relative_to(Path(root).resolve())
+    except ValueError:
+        return VerificationResult(
+            ok=False,
+            error_code=ERROR_DOCUMENT_OUT_OF_ROOT,
+            error_message=f"source_path {source_path!r} resolves outside repo root {root!r}",
+        )
     try:
         document_bytes = doc_path.read_bytes()
```

The fix introduces `ERROR_DOCUMENT_OUT_OF_ROOT` as a new structured error code so callers can distinguish a boundary violation from a generic missing-document error.
