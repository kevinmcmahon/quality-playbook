# BUG-008 — Iteration suggestion printed unconditionally even on failure

## Requirement
REQ-014 (Suggestion suppressed on run failure). `print_suggested_next_command` must not
suggest "next iteration" when the run returned non-zero. On failure, the suggestion must
tell the user to investigate or re-run the failed phase.
Doc source: `[source]` — EXPLORATION.md §F-8.

## Candidate origin
Source stage: Phase 1 exploration / Pass 1 structural review.
Candidate bug ID: CB-8.

## Location
- File: `bin/run_playbook.py`
- Lines: 930–946
- Function(s): `execute_run`, `print_suggested_next_command`

## Symptom
After a Phase 1 crash (worker exits non-zero), the runner prints the normal "next iteration"
suggestion as if the run succeeded. The user is invited to run Phase 2 on a failed Phase 1,
leading to artifacts built on incomplete exploration.

## Root cause
`execute_run` tracks `failures` (count of non-zero exits) but calls
`print_suggested_next_command(args)` unconditionally when `not suppress_suggestion`.
`print_suggested_next_command` has only one parameter (`args`) and cannot branch on failure
state. The failure count is tracked but never threaded into the suggestion logic.
