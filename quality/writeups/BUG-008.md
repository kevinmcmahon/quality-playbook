# Bug Writeup: BUG-008
> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.5.3 — Andrew Stellman
> Date: 2026-04-28 · Project: quality-playbook

## Summary
Phase 2 through Phase 6 prompts still hardcode the flat Copilot install layout, so a repo-root, Claude, or nested-Copilot run can start cleanly and then fail later only because the prompt family lost the documented fallback contract.

## Spec reference
- Requirement: REQ-009
- Spec basis: `quality/REQUIREMENTS.md:287-304`
- Behavioral contract quote: "Phase 2 through Phase 6 prompts must instruct child agents to resolve `SKILL.md` and `references/*.md` through the same four-path fallback list"

## The code
At `bin/run_playbook.py:726-752`, `755-908`, `911-964`, `967-1088`, and `1089-1090`, the later-phase prompt builders still direct children to `.github/skills/SKILL.md` and `.github/skills/references/...` directly. That contradicts the canonical four-path fallback contract published in `SKILL.md:49-58` and the centralized `SKILL_FALLBACK_GUIDE` already used by the entry prompts at `bin/run_playbook.py:613-618`.

## Observable consequence
A run launched from a valid repo-root, Claude, or nested-Copilot install can pass Phase 1 with the right skill file and then be redirected in Phase 2-6 to a flat `.github/skills/` path that does not exist. The result is a success-shaped portability gap: entry is hardened, but the later phases still assume one specific install layout.

## The fix
Refactor `phase2_prompt()` through `phase6_prompt()` to reuse the same fallback-list wording the entry prompts already use, and remove direct `.github/skills/references/...` path literals from later-phase instructions.

```diff
diff --git a/bin/run_playbook.py b/bin/run_playbook.py
--- a/bin/run_playbook.py
+++ b/bin/run_playbook.py
@@
-3. .github/skills/SKILL.md - read the Phase 2 section ...
+3. Use the documented fallback list for SKILL.md: SKILL.md, .claude/skills/quality-playbook/SKILL.md, .github/skills/SKILL.md, .github/skills/quality-playbook/SKILL.md. Resolve cited reference files through the same fallback order.
@@
-5. .github/skills/SKILL.md - read the Phase 3 section ... Also read .github/skills/references/review_protocols.md.
+5. Read the Phase 3 section from the resolved SKILL.md path. Read `references/review_protocols.md` from the matching references directory resolved through the same fallback order.
```
