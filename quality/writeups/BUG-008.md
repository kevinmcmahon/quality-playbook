# BUG-008: quality_gate.sh crashes on macOS bash 3.2 with no arguments

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## 1. Summary

quality_gate.sh crashes with "unbound variable" on macOS (bash 3.2) when invoked with no arguments. The script sets `set -uo pipefail` (line 32) and initializes `REPO_DIRS=()` (line 37). When no arguments are provided, the else branch at line 637 iterates `for name in "${REPO_DIRS[@]}"` over the empty array. On bash 3.2, `"${arr[@]}"` on an empty array with `set -u` triggers a fatal error instead of iterating zero times.

## 2. Spec reference

quality_gate.sh lines 25-27: The script defines exit code 1 for usage errors ("one or more checks failed") and provides a usage message at line 654. The expected behavior for no-arguments invocation is the usage message, not a crash.

## 3. The code

```bash
# quality_gate.sh line 32
set -uo pipefail

# quality_gate.sh line 37
REPO_DIRS=()

# quality_gate.sh lines 637-639
else
    resolved=()
    for name in "${REPO_DIRS[@]}"; do   # <-- crashes here on bash 3.2
```

The `"${REPO_DIRS[@]}"` expansion on an empty array is undefined behavior in bash 3.2 with `set -u`. Bash 4.4+ handles this correctly (expands to nothing).

## 4. Observable consequence

On macOS, running `./quality_gate.sh` with no arguments produces:
```
bash: REPO_DIRS[@]: unbound variable
```
instead of the expected usage message. The script exits with code 127 (command error) rather than code 1.

## 6. The fix

Use the `${arr[@]+"${arr[@]}"}` pattern, which is safe on all bash versions:

```diff
--- a/quality_gate.sh
+++ b/quality_gate.sh
@@ -636,7 +636,7 @@ elif [ ${#REPO_DIRS[@]} -eq 1 ] && [ "${REPO_DIRS[0]}" = "." ]; then
     REPO_DIRS=("$(pwd)")
 else
     resolved=()
-    for name in "${REPO_DIRS[@]}"; do
+    for name in ${REPO_DIRS[@]+"${REPO_DIRS[@]}"}; do
         if [ -d "$name/quality" ]; then
             resolved+=("$name")
```

The `${var+expr}` syntax expands to `expr` only if `var` is set and non-empty, avoiding the unbound variable error on bash 3.2.

## 7. The test

Run `./quality_gate.sh` on macOS (bash 3.2) with no arguments. Without the fix: "unbound variable" error, exit 127. With the fix: "Usage: ./quality_gate.sh ..." message, exit 1.
