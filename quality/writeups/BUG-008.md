# BUG-008: Gate Missing total and verified Summary Key Validation

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (bootstrap self-audit)

## 1. Summary

quality_gate.sh validates 3 of 5 required summary keys in tdd-results.json (confirmed_open, red_failed, green_failed) but does not check `total` or `verified`. The normative text requires all 5 keys.

## 2. Spec reference

SKILL.md line 1394: "The `summary` object must contain exactly these keys: `total`, `verified`, `confirmed_open`, `red_failed`, `green_failed`."

## 3. The code

```
quality_gate.sh:229-235
# Checks: confirmed_open, red_failed, green_failed
# Missing: total, verified
```

## 4. Observable consequence

A tdd-results.json with a summary containing only 3 keys passes the gate. Downstream tooling reading `total` or `verified` gets undefined values.

## 6. The fix

```diff
--- a/quality_gate.sh
+++ b/quality_gate.sh
@@ -229,6 +229,8 @@
 for key in "confirmed_open" "red_failed" "green_failed"; do
+for key in "total" "verified" "confirmed_open" "red_failed" "green_failed"; do
   json_has_key "$tdd_json" "$key" "summary"
```

## 7. The test

```bash
test_bug_008_gate_checks_total_and_verified() {
    local checks_total=$(grep -c '"total"' "$GATE" | grep -v 'total_patches\|total_groups')
    local checks_verified=$(grep -c '"verified"' "$GATE")
    [ "$checks_total" -gt 0 ] && [ "$checks_verified" -gt 0 ]
}
```

RED: Test fails because neither `total` nor `verified` appear in the gate's key check loop.
