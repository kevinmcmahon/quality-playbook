# BUG-014: quality_gate.sh heading detection misses #### BUG-NNN format

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## 1. Summary

BUGS.md headings with four or more hashes (`#### BUG-001`) are silently invisible to all four heading detection patterns in quality_gate.sh lines 173-200. This causes bug_count to be 0, which skips all downstream checks (patches, writeups, TDD logs), and the gate passes despite actual bugs being present in BUGS.md.

## 2. Spec reference

SKILL.md line 1611: "Each confirmed bug must use the heading level `### BUG-NNN`... The `### BUG-NNN` heading is what downstream tools grep for when counting bugs, and what the tdd-results.json `id` field must match."

## 3. The code

quality_gate.sh lines 173-181 check four heading patterns:
- Line 173: `^### BUG-[0-9]+` (correct format)
- Line 175: `^## BUG-[0-9]+` (two-hash wrong format)
- Line 178: `^\*\*BUG-[0-9]+` (bold format)
- Line 180: `^- BUG-[0-9]+` (bullet format)

None of these match `#### BUG-001` because: `^###` requires exactly three hashes before a space, `^##` requires exactly two hashes before a space, and the bold/bullet patterns require `**` or `- ` prefixes.

## 4. Observable consequence

If an AI agent generates BUGS.md with `#### BUG-001: description` headings (which is a plausible markdown formatting error), quality_gate.sh reports bug_count=0, skips all TDD/patch/writeup checks, and the gate exits 0 with "Zero bugs" info messages -- despite the bugs being clearly present in the file.

## 6. The fix

Add detection for four-or-more-hash headings:

```diff
--- a/quality_gate.sh
+++ b/quality_gate.sh
@@ -180,6 +180,9 @@
         bullet_headings=$(grep -cE '^- BUG-[0-9]+' "${q}/BUGS.md" || true)
         bullet_headings=${bullet_headings:-0}
 
+        local deep_headings
+        deep_headings=$(grep -cE '^#{4,} BUG-[0-9]+' "${q}/BUGS.md" || true)
+        deep_headings=${deep_headings:-0}
 
         bug_count=$correct_headings
 
@@ -188,6 +191,7 @@
             [ "$wrong_headings" -gt 0 ] && fail "${wrong_headings} heading(s) use ## instead of ###"
             [ "$bold_headings" -gt 0 ] && fail "${bold_headings} heading(s) use **BUG- format"
             [ "$bullet_headings" -gt 0 ] && fail "${bullet_headings} heading(s) use - BUG- format"
+            [ "$deep_headings" -gt 0 ] && fail "${deep_headings} heading(s) use #### or deeper instead of ###"
```

## 7. The test

Test all four detection patterns against `#### BUG-001: test`. All return 0 matches, confirming the heading is invisible. After the fix, the new `^#{4,} BUG-[0-9]+` pattern detects it and reports it as a wrong-format heading.
