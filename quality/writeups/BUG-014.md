# BUG-014: Dangling Phase 7 Cross-References in SKILL.md

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (bootstrap self-audit)

## 1. Summary

Three locations in SKILL.md (lines 436, 1797, 1821) reference "Phase 7" outside its section definition. Since Phase 7 is absent from the plan overview (BUG-002), these are dangling cross-references pointing to an unreachable phase. The convergence check at line 1797 ("skip to Phase 7") routes agents to a phase they cannot find in the roadmap.

## 2. Spec reference

SKILL.md lines 12-31: Plan overview lists Phases 0-6 only. Cross-references must point to phases in the plan overview.

## 3. The code

```
SKILL.md:436 — references Phase 7 in context of iteration
SKILL.md:1797 — "skip to Phase 7" in convergence check
SKILL.md:1821 — Phase 7 in end-of-phase context
```

## 4. Observable consequence

An agent following the convergence check at line 1797 is told to "skip to Phase 7" but finds no Phase 7 heading when scanning the plan overview. The agent either halts or searches the full document body, breaking the structured phase-by-phase flow.

## 6. The fix

```diff
--- a/SKILL.md
+++ b/SKILL.md
@@ -1797,7 +1797,7 @@
-If convergence is reached, skip to Phase 7.
+If convergence is reached, the playbook is complete.
```

Apply similarly to lines 436 and 1821 — remove or redirect Phase 7 references. The exact fix depends on the Phase 7 disposition decision (BUG-002).
