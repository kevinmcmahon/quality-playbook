# BUG-016: Strategy Count Says "Five" But Only Four Exist

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (bootstrap self-audit)

## 1. Summary

SKILL.md line 210 says "one of five strategies" for iteration runs, but references/iteration.md defines exactly four strategies (gap, unfiltered, parity, adversarial) and SKILL.md line 1879 says "all four strategies." This is a self-contradiction within SKILL.md.

## 2. Spec reference

references/iteration.md: Defines 4 strategies with `## Strategy:` headings: gap, unfiltered, parity, adversarial. SKILL.md line 1879: "all four strategies."

## 3. The code

```
SKILL.md:210
Each iteration uses one of five strategies...
```

But iteration.md defines exactly 4, and line 1879 says "four."

## 4. Observable consequence

A user reading line 210 expects five strategies. When they find only four in iteration.md, they search for the missing fifth strategy — wasting time on a phantom feature. An agent may also attempt to find or hallucinate a fifth strategy.

## 6. The fix

```diff
--- a/SKILL.md
+++ b/SKILL.md
@@ -210,7 +210,7 @@
-Each iteration uses one of five strategies
+Each iteration uses one of four strategies
```
