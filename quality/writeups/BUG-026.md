# BUG-026 — Helper functional-test discovery accepts undocumented `test_functional_test.*`

## 1. Summary

The helper library treats `quality/test_functional_test.py` as if it were a valid functional-test
artifact, even though the published naming contract never authorizes that alias. Because
`final_artifact_gaps()` trusts the helper, the runner can suppress a missing-functional-test report
for a repo that still lacks any documented functional-test filename.

## 2. Spec reference

REQ-027 — Helper functional-test discovery honors only documented names.

> `[Tier 1] [functional_tests.md] §Writing Functional Tests` — "functional_test.go (Go), FunctionalTest.java, FunctionalSpec.scala, functional.test.ts"

## 3. The code

`bin/benchmark_lib.py:19-26` includes `test_functional_test.*` in `FUNCTIONAL_TEST_PATTERNS`.
`bin/benchmark_lib.py:164-169` exposes that set through `find_functional_test()`, and
`bin/run_playbook.py:592-594` uses the helper in `final_artifact_gaps()`. The helper therefore
turns an undocumented alias into a user-visible artifact decision.

## 4. Observable consequence

If a repo contains only `quality/test_functional_test.py`, the runner reports a cleaner end-of-run
artifact list than the published contract allows. A maintainer can believe the functional-test
artifact exists even though the documented naming matrix says otherwise.

## 6. The fix

```diff
--- a/bin/benchmark_lib.py
+++ b/bin/benchmark_lib.py
@@
 FUNCTIONAL_TEST_PATTERNS = (
     "test_functional.*",
     "functional.test.*",
     "functional_test.*",
     "FunctionalTest.*",
     "FunctionalSpec.*",
-    "test_functional_test.*",
 )
```

## 7. The test

`quality/test_regression.py::AdversarialIterationRegressions::test_reg_adv26_final_artifact_gaps_rejects_test_functional_test_alias`
creates a minimal repo with all the other Phase 2 artifacts present but only the undocumented
`test_functional_test.py` alias for functional coverage. On unpatched code,
`final_artifact_gaps()` omits `"functional test"` and the test fails. After the helper tuple is
narrowed to documented names, the same test passes because the missing-artifact report is restored.
