# BUG-015 — _parse_porcelain_path returns quoted paths with surrounding quotes intact

## Requirement
REQ-016 (_parse_porcelain_path handles quoted paths). The parser must strip Git's
surrounding double-quotes from paths with special characters (spaces, unicode, tabs).
Doc source: `[source]` — EXPLORATION.md Notes for Artifact Generation.

## Candidate origin
Source stage: new — Pass 1 structural review (REQ-016).

## Location
- File: `bin/benchmark_lib.py`
- Lines: 185–196
- Function(s): `_parse_porcelain_path`

## Symptom
When `cleanup_repo` encounters a tracked file with a space in its name (e.g.,
`docs_gathered/My Notes.md`), `git status --porcelain` outputs `' M "My Notes.md"'`.
`_parse_porcelain_path` returns `'"My Notes.md"'` with quotes. The subsequent
`git checkout -- '"My Notes.md"'` call fails because the shell sees the literal
quote characters as part of the filename, not as string delimiters.

## Root cause
`_parse_porcelain_path` (lines 185-196) ends with `return rest.strip()`. No quote-stripping
code is present. Git adds surrounding double-quotes to paths containing spaces or special
characters when `--porcelain` is used without `-z` (null-separator) mode. The function does
not account for this quoting convention.
