# BUG-001: quality_gate.sh downgrades 6 required artifacts to WARN, allowing silent gate pass

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## 1. Summary

quality_gate.sh uses `warn()` instead of `fail()` for 6 artifacts that SKILL.md marks as "Required: Yes" in the artifact contract table (lines 86-115). Since the exit code (lines 670-676) only checks `FAIL > 0`, a run missing all six artifacts exits 0 ("GATE PASSED").

## 2. Spec reference

- SKILL.md lines 83-85: "If the gate checks for it, this skill must instruct its creation. This is the canonical list."
- SKILL.md lines 86-115: artifact contract table listing CONTRACTS.md, RUN_CODE_REVIEW.md, RUN_SPEC_AUDIT.md, RUN_INTEGRATION_TESTS.md, RUN_TDD_TESTS.md, and test_functional.* as "Required: Yes"
- Requirements: REQ-001, REQ-002, REQ-006

## 3. Code citation

quality_gate.sh lines 115-129:
- Lines 115-116: Comment references non-existent "BUG-005" as justification for WARN
- Lines 117-123: Loop uses `warn "${f} missing"` for CONTRACTS.md through RUN_TDD_TESTS.md
- Lines 124-129: test_functional.* check uses `warn "test_functional.* missing"`

## 4. Observable consequence

A quality gate run missing CONTRACTS.md, all RUN_*.md files, test_functional.*, and AGENTS.md exits 0 with 0 FAILs and 8 WARNs. The gate falsely reports success for an incomplete quality playbook run.

## 5. Test description

Two regression tests confirm the bug:
- `test_bug_001_required_artifacts_warn_severity`: Removes CONTRACTS.md (a Required: Yes artifact) and verifies the gate exits 1. On unpatched code, gate exits 0 (XFAIL).
- `test_bug_001_all_warn_artifacts_missing`: Removes all 6 WARN-checked artifacts and verifies the gate exits 1. On unpatched code, gate exits 0 (XFAIL).

TDD result: RED (test fails on unpatched code) -> GREEN (test passes after fix).
