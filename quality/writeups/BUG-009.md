# BUG-009: Unguarded Glob in Writeup For Loop

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (bootstrap self-audit)

## 1. Summary

quality_gate.sh line 507 uses a `for` loop with a glob pattern (`quality/writeups/BUG-*.md`) without a `nullglob` guard or `compgen` check. When no writeup files exist, bash iterates the literal glob string, causing a spurious file-not-found error in the loop body.

## 2. Spec reference

Bash best practices: glob patterns in `for` loops require `nullglob` or `compgen` guard to handle the zero-match case.

## 3. The code

```
quality_gate.sh:507
for writeup in quality/writeups/BUG-*.md; do
    # If no files match, $writeup is the literal string "quality/writeups/BUG-*.md"
```

## 4. Observable consequence

On a valid run with no writeups (zero confirmed bugs, or writeups not yet generated), the loop body processes the literal glob string, producing a confusing error message about a non-existent file.

## 6. The fix

```diff
--- a/quality_gate.sh
+++ b/quality_gate.sh
@@ -506,6 +506,7 @@
+shopt -s nullglob
 for writeup in quality/writeups/BUG-*.md; do
```

Or use `compgen`:

```diff
-for writeup in quality/writeups/BUG-*.md; do
+for writeup in $(compgen -G "quality/writeups/BUG-*.md"); do
```

## 7. The test

```bash
test_bug_009_writeup_loop_guarded() {
    local has_guard=$(grep -cE '(nullglob|compgen.*BUG-)' "$GATE")

```
