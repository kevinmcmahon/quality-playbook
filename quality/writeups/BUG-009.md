# Bug Writeup: BUG-009
> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.5.3 — Andrew Stellman
> Date: 2026-04-28 · Project: quality-playbook

## Summary
Repository-side skill detection still uses an outdated fallback tuple and incomplete warning text, so helper discovery and operator-facing installation checks no longer match the published four-path support matrix.

## Spec reference
- Requirement: REQ-010
- Spec basis: `quality/REQUIREMENTS.md:306-323`
- Behavioral contract quote: "Helper discovery, operator-facing warnings, and shipped orchestrator docs must enumerate all four supported SKILL locations in the documented order"

## The code
`bin/benchmark_lib.py:42-47` defines `SKILL_INSTALL_LOCATIONS` as flat Copilot, Claude, repo root, nested Copilot. `detect_repo_skill_version()` and `find_installed_skill()` both walk that tuple unchanged (`bin/benchmark_lib.py:144-164`). `bin/run_playbook.py:592-597` then emits a "No SKILL.md found" warning that lists only three locations and omits the nested Copilot path entirely.

## Observable consequence
Repository-side install detection can treat a flat installed copy as canonical even when the documented first hit is the repo-root skill, and operator-facing warnings can falsely imply that a nested-Copilot-only install is unsupported. The result is incorrect installation guidance before the run even starts.

## The fix
Make the helper tuple and warning message follow one shared canonical order: repo root, Claude, flat Copilot, nested Copilot.

```diff
diff --git a/bin/benchmark_lib.py b/bin/benchmark_lib.py
--- a/bin/benchmark_lib.py
+++ b/bin/benchmark_lib.py
@@
-SKILL_INSTALL_LOCATIONS = (
-    Path(".github") / "skills" / "SKILL.md",
-    Path(".claude") / "skills" / "quality-playbook" / "SKILL.md",
-    Path("SKILL.md"),
-    Path(".github") / "skills" / "quality-playbook" / "SKILL.md",
-)
+SKILL_INSTALL_LOCATIONS = (
+    Path("SKILL.md"),
+    Path(".claude") / "skills" / "quality-playbook" / "SKILL.md",
+    Path(".github") / "skills" / "SKILL.md",
+    Path(".github") / "skills" / "quality-playbook" / "SKILL.md",
+)
```
