# BUG-009: TOOLKIT.md Copilot reference installation path not in SKILL.md fallback chain

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## 1. Summary

TOOLKIT.md (lines 24-26) and agents/quality-playbook.agent.md (lines 29-31) both instruct GitHub Copilot users to install reference files at `.github/skills/references/`. However, SKILL.md's reference file resolution fallback chain (lines 48-51) does not include this path. A user following the documented installation instructions would place reference files where the skill cannot find them, causing the agent to silently proceed without reference files.

## 2. Spec reference

SKILL.md lines 46-53: "When a reference file is mentioned, resolve it by checking these paths in order and using the first one that exists." The four paths listed are: (1) `references/` relative to SKILL.md, (2) `.claude/skills/quality-playbook/references/`, (3) `references/` at repo root, (4) `.github/skills/quality-playbook/references/`. The path `.github/skills/references/` is absent.

## 3. The code

TOOLKIT.md (user-facing installation guide):
```
GitHub Copilot:
  .github/skills/SKILL.md
  .github/skills/references/          <-- installed here
  .github/skills/quality_gate.sh
```

SKILL.md (reference resolution):
```
1. references/                                          <-- relative to SKILL.md
2. .claude/skills/quality-playbook/references/          <-- Claude Code
3. references/                                          <-- repo root
4. .github/skills/quality-playbook/references/          <-- NOT the same as .github/skills/references/
```

Path 1 ("relative to SKILL.md") could theoretically resolve to `.github/skills/references/` if SKILL.md is at `.github/skills/SKILL.md` and the agent resolves relative paths based on the SKILL.md location. However, this depends on agent implementation and is not guaranteed.

## 4. Observable consequence

A Copilot user follows TOOLKIT.md's instructions. The agent reads SKILL.md but cannot find reference files through the four fallback paths. Per REQ-004 (from the baseline), SKILL.md has no explicit error handling when all paths fail. The agent silently proceeds without reading iteration.md, review_protocols.md, spec_audit.md, verification.md, or any other reference file. This produces degraded output with no warning to the user.

## 6. The fix

Add `.github/skills/references/` to SKILL.md's reference resolution fallback chain:

```diff
--- a/SKILL.md
+++ b/SKILL.md
@@ -48,6 +48,7 @@ This skill references files in a `references/` directory
 1. `references/` (relative to SKILL.md — works when running from the skill directory)
 2. `.claude/skills/quality-playbook/references/` (Claude Code installation)
 3. `references/` (GitHub Copilot installation)
-4. `.github/skills/quality-playbook/references/` (alternate Copilot installation)
+4. `.github/skills/references/` (GitHub Copilot flat installation)
+5. `.github/skills/quality-playbook/references/` (alternate Copilot installation)
```

## 7. The test

Verify that `.github/skills/references/` appears in SKILL.md's reference resolution section. Without the fix: `grep -q '.github/skills/references/' SKILL.md` returns no match. With the fix: grep matches.
