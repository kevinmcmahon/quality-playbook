# BUG-017: EXPLORATION.md declared mandatory but missing from artifact contract table

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## 1. Summary

SKILL.md line 259 declares `quality/EXPLORATION.md` as "mandatory in all modes" but the artifact contract table at lines 86-115 -- described as "the canonical list" at line 85 -- does not include it. This creates a contradiction: the canonical list says unlisted artifacts should not be gate-enforced, while the body text says EXPLORATION.md is mandatory. quality_gate.sh correctly follows the table (no EXPLORATION.md check), meaning the mandatory declaration is unenforceable.

## 2. Spec reference

SKILL.md line 85: "This is the canonical list -- any artifact not listed here should not be gate-enforced, and any gate check should trace to an artifact listed here."

SKILL.md line 259: "This file is mandatory in all modes."

These two statements contradict each other regarding EXPLORATION.md.

## 3. The code

SKILL.md lines 86-115 define the artifact contract table. The table contains entries for QUALITY.md, REQUIREMENTS.md, CONTRACTS.md, test_functional.*, RUN_CODE_REVIEW.md, RUN_INTEGRATION_TESTS.md, RUN_SPEC_AUDIT.md, RUN_TDD_TESTS.md, BUGS.md, COVERAGE_MATRIX.md, COMPLETENESS_REPORT.md, PROGRESS.md, AGENTS.md, and various conditional artifacts. EXPLORATION.md does not appear anywhere in the table.

SKILL.md line 259 states: "Write your Phase 1 exploration findings to `quality/EXPLORATION.md` before proceeding. This file is mandatory in all modes."

## 4. Observable consequence

A playbook run that skips Phase 1 entirely and produces no EXPLORATION.md can pass quality_gate.sh, because the gate follows the artifact table (which omits EXPLORATION.md). Since EXPLORATION.md is the foundation for Phase 2 artifact generation, a passing gate does not guarantee that the quality artifacts were derived from actual exploration findings.

## 6. The fix

Add EXPLORATION.md to the artifact contract table:

```diff
--- a/SKILL.md
+++ b/SKILL.md
@@ -100,6 +100,7 @@
 | Completeness report | `quality/COMPLETENESS_REPORT.md` | Yes | Phase 2 (baseline), Phase 5 (final verdict) |
 | Progress tracker | `quality/PROGRESS.md` | Yes | Throughout |
+| Exploration findings | `quality/EXPLORATION.md` | Yes | Phase 1 |
 | AI bootstrap | `AGENTS.md` | Yes | Phase 2 |
 | Bug writeups | `quality/writeups/BUG-NNN.md` | If bugs found | Phase 5 |
```

After applying the fix patch, the test confirms the table includes EXPLORATION.md, resolving the contradiction.
