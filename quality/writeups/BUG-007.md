# BUG-007: Version detection path asymmetry between global and per-repo checks

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## 1. Summary

Global version detection (lines 59-67) tries 6 paths including `${SCRIPT_DIR}/../SKILL.md` and `${SCRIPT_DIR}/SKILL.md`. Per-repo version stamp validation (lines 581-586) tries only 4 paths (relative to `repo_dir`). If SKILL.md is located only via a SCRIPT_DIR-relative path, the global version is detected but per-repo version stamp validation silently skips.

## 2. Spec reference

- quality_gate.sh internal consistency
- SKILL.md lines 908-922: version stamp consistency requirement
- Requirement: REQ-008

## 4. Observable consequence

When quality_gate.sh is run from a directory where SKILL.md is found only via SCRIPT_DIR-relative paths, the global version is detected correctly, but per-repo version stamp checks are silently skipped (line 601: `warn "Cannot detect skill version from SKILL.md"`). Version stamp mismatches go undetected.
