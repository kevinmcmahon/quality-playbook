# BUG-007: Artifact Table verify.sh Phase Assignment Contradicts Instructions

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (bootstrap self-audit)

## 1. Summary

The artifact contract table (SKILL.md line 97) says `quality/mechanical/verify.sh` is "Created In: Phase 5." But the Phase 2a instructions (line 555) have an "Immediate integrity gate (mandatory, Phase 2a)" that requires verify.sh to already exist. An agent following the table will not create verify.sh until Phase 5, causing the Phase 2a gate to fail.

## 2. Spec reference

SKILL.md line 555: "Immediate integrity gate (mandatory, Phase 2a)."

The behavioral contract: the "Created In" column must match the earliest phase requiring the artifact.

## 3. The code

```
SKILL.md:97
| Mechanical verify script | `quality/mechanical/verify.sh` | Yes (benchmark) | Phase 5 |
```

But line 555 requires it in Phase 2a.

## 4. Observable consequence

An agent consulting the artifact table skips verify.sh creation until Phase 5. When it reaches the Phase 2a integrity gate, verify.sh does not exist, and the gate either fails or is silently skipped.

## 6. The fix

```diff
--- a/SKILL.md
+++ b/SKILL.md
@@ -97,7 +97,7 @@
-| Mechanical verify script | `quality/mechanical/verify.sh` | Yes (benchmark) | Phase 5 |
+| Mechanical verify script | `quality/mechanical/verify.sh` | Yes (benchmark) | Phase 2 |
```
