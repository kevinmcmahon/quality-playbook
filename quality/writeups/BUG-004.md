# BUG-004: Gate Does Not Validate TDD Log Status Tags Despite SKILL.md Claim

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (bootstrap self-audit)

## 1. Summary

SKILL.md line 1137 claims quality_gate.sh "will check for its presence" when describing TDD log status tags (RED/GREEN/NOT_RUN/ERROR). The gate only checks whether the log files exist as files on disk — it does not read the first line to validate the status tag. The specification makes a false claim about gate enforcement.

## 2. Spec reference

SKILL.md line 1137: "quality_gate.sh will check for its presence."

The behavioral contract: the gate validates TDD log content (status tags), or the specification does not claim it does.

## 3. The code

```
quality_gate.sh:277-322 (TDD log checks)
# Checks file existence only: -f "$log_file"
# No head -1 / sed -n 1p to read status tag
```

## 4. Observable consequence

A log file containing arbitrary content (e.g., just whitespace or an error message) passes the gate. The TDD Log Closure Gate in Phase 5 (SKILL.md lines 1585-1589) specifies that "the first line of each log file is one of: RED, GREEN, NOT_RUN, ERROR" — but this is only enforced by the Phase 5 instructions, not by the mechanical gate.
