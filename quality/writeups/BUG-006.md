# BUG-006 — Phase 3 gate checks only 4 of 9 required Phase 2 artifacts

## Requirement
REQ-010 (Phase 3/4/5 entry gates require all Phase 2 / prior-phase artifacts). The Phase 3
entry gate must require every Phase 2 artifact marked "Required" in the SKILL.md artifact
contract.
Doc source: `[source]` — EXPLORATION.md §F-5.

## Candidate origin
Source stage: Phase 1 exploration / Pass 1 structural review.
Candidate bug ID: CB-6.

## Location
- File: `bin/run_playbook.py`
- Lines: 459–463
- Function(s): `check_phase_gate`

## Symptom
A partial Phase 2 run that produces only `REQUIREMENTS.md`, `QUALITY.md`, `CONTRACTS.md`,
and `RUN_CODE_REVIEW.md` is allowed into Phase 3. The five missing artifacts
(`COVERAGE_MATRIX.md`, `COMPLETENESS_REPORT.md`, `RUN_INTEGRATION_TESTS.md`,
`RUN_SPEC_AUDIT.md`, `RUN_TDD_TESTS.md`) are not detected. Phase 3 artifacts then lack
coverage traceability and integration test protocols.

## Root cause
The Phase 3 gate at `run_playbook.py:459-463` hard-codes only four filenames:
`["REQUIREMENTS.md", "QUALITY.md", "CONTRACTS.md", "RUN_CODE_REVIEW.md"]`. Phase 2
produces nine required `.md` files plus `test_functional.py`. The gate was not updated
when the Phase 2 artifact contract was extended.
