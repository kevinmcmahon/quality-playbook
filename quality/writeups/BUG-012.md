# BUG-012: review_protocols.md.orig stale backup file in references/

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.4.0 — Andrew Stellman
> Date: 2026-04-15 · Project: quality-playbook (QFB-bootstrap)

## 1. Summary

A stale `.orig` backup file exists at `references/review_protocols.md.orig` (611 lines). It is not documented in SKILL.md's reference file table but will be read by agents instructed to "read every file in the references/ directory." The file contains different content than the current `review_protocols.md`, including different phase numbering that could confuse agents.

## 2. Spec reference

SKILL.md lines 2075-2083 list all reference files. `review_protocols.md.orig` is not among them. agents/quality-playbook.agent.md line 41 instructs: "read SKILL.md and every file in the `references/` directory."

## 3. The code

`references/review_protocols.md.orig` is a 611-line file. The current `review_protocols.md` has different content. Notably, the .orig file has "Phase 3: Results" at line 376 while the current file has "Phase 6: Results" (BUG-004). An agent reading both files would encounter conflicting phase numbering.

## 4. Observable consequence

An AI agent following the instruction to "read every file in references/" will read both `review_protocols.md` and `review_protocols.md.orig`. The conflicting content -- particularly the different phase numbering -- could cause the agent to use stale instructions or merge conflicting information when generating quality artifacts.

## 6. The fix

Delete the stale backup file:

```diff
--- a/references/review_protocols.md.orig
+++ /dev/null
@@ -1,611 +0,0 @@
-# Review Protocols (Files 3 and 4)
-... (611 lines removed)
```

## 7. The test

The regression test checks whether `review_protocols.md.orig` exists in the references/ directory AND is not documented in SKILL.md. On unpatched code, the file exists and is undocumented, confirming the bug.
