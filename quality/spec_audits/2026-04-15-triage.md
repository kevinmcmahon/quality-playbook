### SA-07 deep analysis: verification.md benchmark 40

This is a spec inconsistency, not a code bug. The benchmark document lists a subset of required files. The fix is to update verification.md, not quality_gate.sh. However, since this audit is running on the QFB-bootstrap repo itself (which IS the quality-playbook skill), this is a **spec bug** — the verification.md reference file is a normative document that agents read. An agent relying on benchmark 40 to determine what files to check would miss 7 required artifacts.

**Determination: Spec inconsistency.** Tracked as a spec-audit finding but not promoted to a code BUG because the defect is in the specification, not executable code.

### SA-08 deep analysis: Version detection path asymmetry

The global version detection (lines 59-67) tries:
1. `${SCRIPT_DIR}/../SKILL.md`
2. `${SCRIPT_DIR}/SKILL.md`
3. `SKILL.md`
4. `.claude/skills/quality-playbook/SKILL.md`
5. `.github/skills/SKILL.md`
6. `.github/skills/quality-playbook/SKILL.md`

The per-repo version stamp check (lines 581-586) tries:
1. `${repo_dir}/SKILL.md`
2. `${repo_dir}/.claude/skills/quality-playbook/SKILL.md`
3. `${repo_dir}/.github/skills/SKILL.md`
4. `${repo_dir}/.github/skills/quality-playbook/SKILL.md`

The per-repo check is missing `${SCRIPT_DIR}/../SKILL.md` and `${SCRIPT_DIR}/SKILL.md`. This means: if quality_gate.sh is run from a directory where SKILL.md is found via `${SCRIPT_DIR}/../SKILL.md` (the most common path when the script is in the skill directory), the global version is detected, but the per-repo version stamp check would fail to find SKILL.md and skip version validation entirely (line 601: `warn "Cannot detect skill version from SKILL.md"`).

This is a real behavioral divergence, but it has limited practical impact: when running `quality_gate.sh .` from the repo root, `${repo_dir}/SKILL.md` resolves correctly. The asymmetry only matters when quality_gate.sh is run from a different directory on a repo that has SKILL.md only via SCRIPT_DIR-relative paths.

**Determination: Real divergence with limited impact.** Promoted to BUG-007 (LOW severity) because the path asymmetry could silently skip version stamp validation.

### SA-11 deep analysis: mechanical/ directory handling

SKILL.md line 110 says "Required: Yes (benchmark)". quality_gate.sh lines 498-512 treat the absence of `mechanical/` as INFO. PROGRESS.md for this project says "Mechanical verification: NOT APPLICABLE -- no dispatch/registry/enumeration contracts in scope."

The artifact table says "Yes (benchmark)" unconditionally, but the broader specification (SKILL.md lines 525-540) explains that mechanical verification is only needed when dispatch-function contracts exist. The gate correctly does not fail when no mechanical/ directory exists, because many projects have no dispatch contracts.

**Determination: Design choice.** The artifact table entry "Yes (benchmark)" is misleading — it should say "Yes (benchmark, if dispatch contracts exist)" or similar. This is a spec clarity issue, not a code bug.
