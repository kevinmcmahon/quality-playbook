### quality_gate.sh

- Spec says: All JSON field presence checks within the same validation section should use the same helper function for consistency. The parity exploration identified that `json_key_count` uses a colon-anchored pattern (stronger) while `json_has_key` uses a bare key-name match (weaker).
- Code does: Uses `json_has_key` for summary keys (`total`, `verified`, `confirmed_open`, `red_failed`, `green_failed`) and `json_key_count` for per-bug fields. This is an inconsistency in the same code section.
