**Requirement confidence tiers:**
Requirements are tagged with `[Req: tier — source]`. Weight your findings by tier:
- **formal** — written in SKILL.md or quality_gate.sh. Authoritative. Divergence is a real finding.
- **user-confirmed** — stated in README or TOOLKIT.md but not in SKILL.md. Treat as authoritative.
- **inferred** — deduced from code behavior. Lower confidence. Report as NEEDS REVIEW.

**Rules:**
- ONLY list defects. Do not summarize what matches.
- For EVERY defect, cite specific file and line number(s).
  If you cannot cite a line number, do not include the finding.
- Before claiming missing, grep the codebase.
- Before claiming exists, read the actual file content.
- Classify each finding: MISSING / DIVERGENT / UNDOCUMENTED / PHANTOM
- For findings against inferred requirements, add: NEEDS REVIEW

**Defect classifications:**
- **MISSING** — Spec/requirement says it should exist, code doesn't implement it
- **DIVERGENT** — Both requirement and code address it, but they disagree
- **UNDOCUMENTED** — Code does it, specification doesn't describe it
- **PHANTOM** — Specification describes it, but it's actually implemented differently

### Confidence levels

| Confidence | Found By | Action |
|------------|----------|--------|
| Highest | All three | Almost certainly real — confirm and add to BUGS.md |
| High | Two of three | Likely real — verify with probe and confirm |
| Needs verification | One only | Deploy verification probe — do not dismiss by default |

### Minority finding rule

Any finding where only one of three auditors flags it requires a re-investigation. Read the specific code location and make an explicit CONFIRMED/FALSE-POSITIVE determination.

### Verification probes must produce executable evidence

For each confirmation or rejection, write an assertion in `quality/spec_audits/triage_probes.sh`:

```bash
#!/bin/bash
# Triage verification probes — executable evidence for audit findings

# Probe 1: tdd_verified vs verified (REQ-001)
if grep -q '"tdd_verified"' SKILL.md; then
    echo "CONFIRMED: SKILL.md contains 'tdd_verified' (line $(grep -n '"tdd_verified"' SKILL.md | head -1 | cut -d: -f1))"
else
    echo "REJECTED: No 'tdd_verified' in SKILL.md"
fi

# Probe 2: Phase 7 in plan overview (REQ-002)
PLAN_PHASE7=$(sed -n '12,31p' SKILL.md | grep -ci 'phase 7' || echo 0)
if [ "$PLAN_PHASE7" -eq 0 ]; then
    echo "CONFIRMED: Phase 7 not in plan overview (lines 12-31)"
else
    echo "REJECTED: Phase 7 IS in plan overview"
fi

# [Add probes for each finding...]
```

### Categorize Each Confirmed Finding

- **Spec bug** — Spec is wrong, code is fine → update spec
- **Design decision** — Human judgment needed → discuss and decide
- **Real code bug** — Fix in small batches by subsystem
- **Documentation gap** — Feature exists but undocumented → update docs
- **Missing test** — Code is correct but no test verifies it → add test
- **Inferred requirement wrong** — The inferred requirement doesn't match intent → correct it
