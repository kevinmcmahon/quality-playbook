**What REQ-001 claims:** json_has_key must verify the key appears as an actual JSON key (preceding `:`), not merely as a substring.

**Analysis:** BUG-H1 (fixing json_has_key to add colon anchor) would automatically fix the REQ-022 violation at lines 259-265 as a side effect. The two requirements are mutually reinforcing — fixing BUG-H1 fixes the parity gap too. This is a consistency relationship: REQ-022's fix depends on REQ-001's fix.
**Impact:** If BUG-H1 is fixed (json_has_key gets colon anchor), lines 259-265 using json_has_key become correct (the function now enforces the required pattern). If BUG-H1 is fixed but lines 259-265 are separately changed to json_key_count, both paths are correct. If BUG-H1 is NOT fixed but lines 259-265 are changed to json_key_count, the per-bug check is correct but json_has_key remains vulnerable elsewhere. The lowest-risk fix: fix BUG-H1 first, then optionally unify all field checks to json_key_count for clarity.

### Shared Concept: Phase gate entry/exit parity

**Requirements:** REQ-024 (new — Phase 5 entry gate), REQ-003 (existing — Phase 2 entry gate enforces Phase 1 checks)
**What REQ-024 claims:** Phase 5 must have an entry gate verifying Phase 4 artifacts.
**What REQ-003 claims:** Phase 2 entry gate must enforce all Phase 1 completion gate checks.

**Consistency:** INCONSISTENT (structurally)
**Code evidence:**
- SKILL.md:897-907: Phase 2 entry gate with 6 explicit checks on EXPLORATION.md content
- SKILL.md:1573-1590: Phase 5 with no entry gate — immediately proceeds to BUG tracker re-read
- BUG-M3 confirmed that Phase 2 entry gate is ITSELF incomplete (only 6 of 12 checks)

**Analysis:** The phase gate entry/exit pattern is inconsistently applied throughout SKILL.md. Phase 2 has a (partial) entry gate. Phase 5 has no entry gate. BUG-M3 shows even the existing Phase 2 entry gate is deficient. REQ-003 and REQ-024 together describe a system where ALL phases should have complete entry gates. Currently, only Phase 2 has an entry gate, and it's incomplete. The required pattern is: Phase N's exit gate should match Phase (N+1)'s entry gate exactly.
**Impact:** An agent running Phase 5 with no Phase 4 artifacts produces a false-passing Phase 5 (except for the terminal gate backstop at the end). This is a spec-level fix for SKILL.md.
