## Working Directory

Run all commands from the repository root.

## Safety Constraints

- Do not modify source files during integration execution.
- Only create or overwrite files under `quality/results/`.
- Continue past a failed group unless the failure blocks every downstream group.
- This repo has no external services in scope; the integration surface is the real Python runner/gate behavior plus the generated Phase 2 artifacts.

## JUnit XML capture

For each group, also emit a JUnit XML file using pytest's native reporter so the run has machine-readable receipts in addition to the `.log` files:

```bash
pytest quality/test_functional.py -q --junitxml quality/results/integration-group-1.junit.xml -k 'test_req_002_reference_docs_is_authoritative_for_phase1 or test_req_008_direct_and_harness_bootstrap_share_docs_surface'
pytest quality/test_functional.py -q --junitxml quality/results/integration-group-2.junit.xml -k 'test_req_003_phase2_gate_requires_exact_sections or test_req_004_live_index_preserves_project_type'
pytest quality/test_functional.py -q --junitxml quality/results/integration-group-3.junit.xml -k 'test_req_001_skill_entry_points_are_synchronized or test_req_006_installed_gate_retains_citation_verifier'
pytest quality/test_functional.py -q --junitxml quality/results/integration-group-4.junit.xml -k 'test_req_005_cleanup_without_pid_files_stays_run_scoped'
pytest quality/test_functional.py -q --junitxml quality/results/integration-group-5.junit.xml -k 'test_req_007_bootstrap_docs_match_live_version_and_terms'
```

For groups 6 and 7, if you cannot produce framework-native JUnit XML from the shell command itself, set `junit_available` to `false`, keep the `.log` receipt, and note the limitation in `groups[].notes`.

```bash
pytest quality/test_functional.py --collect-only -q -k 'test_req_002_reference_docs_is_authoritative_for_phase1 or test_req_008_direct_and_harness_bootstrap_share_docs_surface'
```

## Post-write validation

After writing `quality/results/integration-results.json`, reopen it and confirm:

1. required root keys (`schema_version`, `skill_version`, `date`, `project`, `recommendation`, `groups`, `summary`, `uc_coverage`) are present,
2. every `groups[]` row contains the documented per-group fields,
3. `recommendation`, `groups[].result`, and `uc_coverage` values use only the allowed enums,
4. no undocumented root keys exist.
