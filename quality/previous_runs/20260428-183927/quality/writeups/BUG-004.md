## Summary
`kill_recorded_processes()` widens missing-PID cleanup into workstation-wide `pkill -f` patterns, so a recovery path for one playbook run can terminate unrelated Claude or Copilot sessions on the same machine.

## Spec reference
- Requirement: REQ-005
- Spec basis: `quality/REQUIREMENTS.md:192-208`
- Behavioral contract quote: "Missing PID files do not trigger a workstation-wide pattern kill."

## The code
At `bin/run_playbook.py:2836-2865`, `_pkill_fallback()` defines generic match strings like `claude -p` and `gh copilot -p`. `kill_recorded_processes()` then calls that fallback whenever `discover_pid_files()` returns no files, even though the safe cleanup contract is supposed to stay scoped to playbook-owned workers.

## Observable consequence
If a crash or partial cleanup removes PID files, the next recovery attempt no longer targets only run-owned processes. Instead, the code sweeps the workstation for generic agent command lines and can kill unrelated interactive sessions that happen to match those substrings.

## The fix
Refuse broad fallback cleanup when the run has no PID files, and tell the operator to intervene manually instead of guessing process ownership from command-line fragments.

