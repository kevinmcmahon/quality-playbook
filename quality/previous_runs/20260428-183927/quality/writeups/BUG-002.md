# Bug Writeup: BUG-002
> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.5.3 — Andrew Stellman
> Date: 2026-04-28 · Project: quality-playbook

## Summary
`check_phase_gate(..., "2")` in `bin/run_playbook.py` accepts a long placeholder `quality/EXPLORATION.md` and unlocks downstream phases without enforcing the written exploration contract.

## Spec reference
- Requirement: REQ-003
- Spec basis: `SKILL.md:1026-1042`; `quality/REQUIREMENTS.md:149-167`
- Behavioral contract quote: "A 120-line placeholder without the required headings never unlocks Phase 2."

## The code
At `bin/run_playbook.py:1158-1165`, the Phase 2 gate checks only whether `quality/EXPLORATION.md` exists and has at least 120 lines. It never verifies the required section headings, pattern deep dives, source-stage labels, or Phase 1 completion marker that the written contract names.

## Observable consequence
If an exploration artifact contains enough filler lines but none of the required headings or content-shape checks, the runner still returns `ok=True` and later phases proceed as if Phase 1 completed correctly. That means Phase 2, Phase 3, and Phase 4 can consume a malformed baseline artifact without any gate failure.

## The fix
Extend the executable Phase 2 gate so it reads the artifact text, checks the required headings and content-shape markers, and refuses to advance unless Phase 1 is actually marked complete.
