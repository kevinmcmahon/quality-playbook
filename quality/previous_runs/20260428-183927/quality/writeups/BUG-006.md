## Summary
The curated bootstrap docs in `docs_gathered/01_README_project.md` and `docs_gathered/29_improvement_axes_and_version_history.md` still present older releases and the legacy `docs_gathered/` intake path as current, so self-audit runs start from stale product framing.

## Spec reference
- Requirement: REQ-007
- Spec basis: `quality/REQUIREMENTS.md:229-245`
- Behavioral contract quote: "Curated bootstrap docs that name the current release use the live root skill version."

## The code
At `docs_gathered/01_README_project.md:1-19`, the curated bootstrap README still says `**Version:** 1.5.0` and tells users to create `docs_gathered/`. At `docs_gathered/29_improvement_axes_and_version_history.md:68-72`, the version-history synthesis still labels v1.5.1 as current. The live repo advertises v1.5.3 and `reference_docs/` instead.

## Observable consequence
When QPB audits itself using the curated bootstrap packet, the Phase 1 context includes outdated release and installation claims. That stale framing can seed requirements and audit reasoning with obsolete version and docs-path assumptions before the code review even starts.

## The fix
Refresh the curated bootstrap packet so the current-version and current-path claims match the live repo's v1.5.3 / `reference_docs/` surface.
