## Contract Concordance

### Contract: Phase 1 EXPLORATION.md line-count threshold

- `SKILL.md:906` — "at least **120** lines of substantive content"
- `SKILL.md:916` — "contains at least **120** lines of substantive content" (gate self-check item 1)
- `orchestrator_protocol.md:41` — "more than **80** lines of substantive content"
- `run_playbook.py:456` — `if line_count < 80` (runner threshold)

**FINDING:** Three-way inconsistency. The canonical spec (`SKILL.md`) mandates 120 everywhere. The reference file (`orchestrator_protocol.md`) says 80. The runner implementation also uses 80. An agent consulting `orchestrator_protocol.md` will apply an 80-line bar; the skill's own gate self-check uses 120. This creates two reachable outcomes for an identical EXPLORATION.md with 85-119 lines: the orchestrator reports "Phase 1 complete", but if the agent then runs the skill's Phase 1 gate self-check, it would fail. **Resolution: orchestrator_protocol.md must be updated to 120 (spec fix, companion to BUG-003).**

### Contract: Version parsing (what the version parser must accept)

- `SKILL.md:1-10` (frontmatter) — `version: 1.4.5` (bare form, no bold)
- `SKILL.md:7` note — "12 inline occurrences of '1.4.5' exist in this file … both bare and bold form are used"
- `CONTRACTS.md C-1 Invariant` — "All three parsers must return the same string … across the grammar SKILL.md itself permits (bare `version:`, `**Version:**` bold, case variations, optional quoting)"
- `benchmark_lib.py:37` — `VERSION_PATTERN = re.compile(r"^\s*(?:version:|\*\*Version:\*\*)\s*...")` handles both
- `benchmark_lib.py:106` — `stripped.startswith("version:")` — handles bare only (case-sensitive)
- `quality_gate.py:183` — `if "version:" in line` — substring, no anchor, no bold

**FINDING:** The C-1 Invariant requires all three parsers to agree. They do not. `benchmark_lib._read_version` (via `VERSION_PATTERN`) handles bold form; `benchmark_lib.skill_version` does not; `quality_gate.detect_skill_version` uses substring with no anchor. Three parsers, three different grammars. Already BUG-001 and BUG-013. The concordance confirms these are not independent bugs — they are manifestations of the same C-1 invariant violation, and a partial fix that addresses only one parser leaves the others non-compliant.

### Contract: archive_previous_run atomicity and control_prompts handling

- `CONTRACTS.md C-14 Statement` — "moves `quality/` to `previous_runs/<timestamp>/quality/` and **deletes** `control_prompts/`"
- `CONTRACTS.md C-14 Invariant` — "`control_prompts/` **must be preserved for debugging**"
- `run_playbook.py:576` — `shutil.rmtree(control_prompts_dir, ignore_errors=True)` (deletes)
- `REQ-009 Conditions of satisfaction (3)` — "`previous_runs/TIMESTAMP/control_prompts/` exists and is a complete copy (not deleted)"

**FINDING:** `CONTRACTS.md C-14` contains an internal contradiction. The Statement says "deletes control_prompts/" — describing the current (buggy) code behavior — while the Invariant says it "must be preserved." The Statement is documenting a known-wrong behavior instead of the contract's intended behavior. The correct Statement should say "moves … and archives `control_prompts/`." This is a spec documentation quality issue — the Invariant and REQ-009 agree on the correct behavior, but the Statement misleads readers into thinking deletion is the contract. **Resolution: update C-14 Statement to say "archives" rather than "deletes" (companion spec fix to BUG-004).**

## Probes

### Probe S-A: Three-way version parser concordance test

- **Hypothesis:** `benchmark_lib._read_version` and `benchmark_lib.skill_version` return the same value for a SKILL.md that uses bold form `**Version:** 1.4.5`.
- **Input:** Synthetic SKILL.md content: `"---\n**Version:** 1.4.5\n---\n"`.
- **Observed:** `_read_version` uses `VERSION_PATTERN.match(line)` — matches bold form → returns `"1.4.5"`. `skill_version` uses `stripped.startswith("version:")` on `"**Version:** 1.4.5"` — does NOT start with `"version:"` → returns `None`.
- **Conclusion:** Confirmed. The two functions in the same module return different results for the same input. C-1 Invariant violated.

```python
# Confirmation proof: skill_version fails on bold form
import re, io
PATTERN = re.compile(r"^\s*(?:version:|\*\*Version:\*\*)\s*([0-9]+(?:\.[0-9]+)+)\b", re.IGNORECASE)
bold_line = "**Version:** 1.4.5\n"
# _read_version path:
assert PATTERN.match(bold_line) is not None, "_read_version handles bold form"
# skill_version path (stripped.startswith):
stripped = bold_line.strip()
assert not stripped.startswith("version:"), "skill_version misses bold form — BUG-001 confirmed"
```
