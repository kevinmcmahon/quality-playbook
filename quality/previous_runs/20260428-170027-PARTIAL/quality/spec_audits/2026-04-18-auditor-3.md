### Probe O-B: validate_iso_date behavior on datetime input

- **Hypothesis:** `validate_iso_date("2026-04-18T23:43:14Z")` returns `"valid"`.
- **Input:** Call `validate_iso_date("2026-04-18T23:43:14Z")` as implemented at `quality_gate.py:156-173`.
- **Observed:** The function calls `re.fullmatch(r"\d{4}-\d{2}-\d{2}", "2026-04-18T23:43:14Z")` which does NOT match (full-match fails — extra characters after the date). Returns `"bad_format"`.
- **Conclusion:** Confirmed. BUG-014.

```python
# Confirmation proof: datetime string produces bad_format
import re
from datetime import date
def validate_iso_date_original(date_str):
    if not date_str:
        return "empty"
    if date_str in ("YYYY-MM-DD", "0000-00-00"):
        return "placeholder"
    if not re.fullmatch(r"\d{4}-\d{2}-\d{2}", date_str):
        return "bad_format"
    return "valid"

result = validate_iso_date_original("2026-04-18T23:43:14Z")
assert result == "bad_format", f"Expected bad_format, got {result!r}"
# Confirms BUG-014
```
