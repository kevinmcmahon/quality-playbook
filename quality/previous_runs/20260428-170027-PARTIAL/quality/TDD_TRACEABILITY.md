## Green-Phase Deferral Rationale

Per `quality/RUN_TDD_TESTS.md §Bootstrap-specific notes`:

> In a bootstrap self-audit, the green phase is intentionally deferred. The QPB audits its
> own source code — applying fix patches to that source would change the working tree in ways
> that could interfere with the ongoing quality analysis. Fixes are proposed as patches in
> `quality/patches/BUG-NNN-fix.patch` for the next release cycle. Phase 5 records verdicts as
> `confirmed open` rather than `TDD verified`.

This deferral is an explicit design choice for bootstrap runs, not an omission. All fix patches
exist on disk and are referenced in `tdd-results.json`. The next maintainer release cycle
applies these patches, runs the regression suite, verifies green phases, and changes verdicts
to `TDD verified`.
