# Mechanical verification

Extraction-based cross-checks for the closed sets that appear throughout the Phase 2 artifacts. These checks re-derive the closed sets directly from source (`bin/benchmark_lib.py`, `bin/run_playbook.py`, `.github/skills/quality_gate/quality_gate.py`) and compare them against what CONTRACTS.md / REQUIREMENTS.md / COVERAGE_MATRIX.md claim about the **current** state of the code.

## What this verifies

- **SKILL_INSTALL_LOCATIONS** — the search list used by `skill_version()` for the bare-name fallback. CONTRACTS.md C-2 claims 3 current entries. Phase 2 docs must match the tuple literal in source.
- **PROTECTED_PREFIXES** — the cleanup safe list. CONTRACTS.md C-3 claims 4 entries; REQ-006 wants AGENTS.md added.
- **ALL_STRATEGIES** — the iteration strategy enum. CONTRACTS.md C-4 claims 4 entries (gap, unfiltered, parity, adversarial).
- **Phase identifiers** — CONTRACTS.md C-5 claims 6 phases ("1".."6").
- **Verdict enum** — CONTRACTS.md C-8 claims 5 allowed verdicts.

Any divergence between the live source value and the documented claim is a FAIL — it means a Phase 2 artifact describes the code incorrectly and must be corrected before Phase 3 begins.

## How to run

```bash
# From the QPB root:
bash quality/mechanical/verify.sh
# Exit code 0 = all extractions match their documented claims.
# Exit code != 0 = at least one divergence. Receipt in quality/results/mechanical-verify.log.
```

Receipts land in:

- `quality/results/mechanical-verify.log` — full transcript, one line per check.
- `quality/results/mechanical-verify.exit` — just the exit code (for scripted readers).

## Files

- `verify.sh` — top-level driver; invokes each Python extractor in order and aggregates pass/fail.
- `extract_install_locations.py` — SKILL_INSTALL_LOCATIONS tuple literal, parsed via `ast`.
- `extract_protected_prefixes.py` — PROTECTED_PREFIXES tuple literal.
- `extract_strategies.py` — ALL_STRATEGIES list literal from `run_playbook.py`.
- `extract_phases.py` — phase-id set literal at `run_playbook.py:161`.
- `extract_verdict_enum.py` — `allowed_verdicts` set literal in `quality_gate.py`.

Each extractor is self-contained: it reads the source file, walks the AST (or matches a regex for non-module-level literals), compares to a hard-coded expected value derived from the Phase 2 artifacts, and exits 0 on match / 1 on divergence.
