## Re-investigation finding A — helper functional discovery accepts an undocumented alias

- **Original source:** adjacent-code follow-up from parity BUG-024 / BUG-025 plus REQ-017 thin evidence.
- **Previous conclusion challenged:** earlier iterations treated `benchmark_lib.FUNCTIONAL_TEST_PATTERNS` as authoritative and focused on making the gate match it.
- **Fresh evidence:**
  1. `bin/benchmark_lib.py:19-26` includes `test_functional_test.*`.
  2. `bin/benchmark_lib.py:164-169` exposes that set through `find_functional_test()`.
  3. `bin/run_playbook.py:592-594` uses `find_functional_test()` in `final_artifact_gaps()`, so this helper decides whether the runner reports the functional-test artifact as missing.
  4. `bin/benchmark_lib.py:284-308` uses the same helper in `build_summary_rows()`, so summaries also inherit the permissive alias.
  5. The published contract is narrower: `SKILL.md:72` and `.github/skills/references/functional_tests.md:3` enumerate `test_functional.py`, `FunctionalSpec.scala`, `FunctionalTest.java`, `functional.test.ts`, and `functional_test.go`. Neither source includes `test_functional_test.*`.
- **Code-path trace:** undocumented filename → `find_functional_test()` returns a truthy path → `final_artifact_gaps()` omits `"functional test"` from the missing-artifact list even though no documented functional-test artifact exists.
- **Observable semantic drift:** the runner can report a playbook artifact set as complete when the only functional test file uses a filename the spec never authorizes.
- **New determination:** **CONFIRMED code bug**. Promote as **BUG-026**.

## Re-investigation finding B — helper regression discovery overstates regression coverage

- **Original source:** DC-parity-01 re-read with the lower adversarial evidentiary bar.
- **Previous conclusion challenged:** earlier parity work assumed the gate might need to broaden regression-name acceptance to match helper-library behavior.
- **Fresh evidence:**

- **Observable semantic drift:** benchmark and artifact summaries can overstate bug-closure readiness for repos that would still FAIL the documented artifact contract.
