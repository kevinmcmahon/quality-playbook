# BUG-002 — SKILL_INSTALL_LOCATIONS missing fourth documented path

## Requirement
REQ-002 (SKILL_INSTALL_LOCATIONS covers four paths). `SKILL_INSTALL_LOCATIONS` in
`benchmark_lib.py` must contain all four paths documented in SKILL.md §Locating reference
files (lines 48-55), in the order SKILL.md specifies.
Doc source: `[SKILL.md §Locating reference files, lines 48-55]`.

## Candidate origin
Source stage: Phase 1 exploration / Pass 1 structural review.
Candidate bug ID: CB-2.

## Location
- File: `bin/benchmark_lib.py`
- Lines: 39–43
- Function(s): module-level constant `SKILL_INSTALL_LOCATIONS`

## Symptom
Users who install QPB at `.github/skills/quality-playbook/SKILL.md` (the alternate Copilot
layout) get a spurious "no SKILL.md found" warning. The runner cannot detect the installed
skill version, which cascades to BUG-001's version-append fallback returning None.

## Root cause
`SKILL_INSTALL_LOCATIONS` is a 3-tuple. The fourth path,
`Path(".github") / "skills" / "quality-playbook" / "SKILL.md"`, is absent. SKILL.md:48-55
documents four paths; `quality_gate.py:969-976` encodes all four. The library was not
updated when the fourth install location was added.
