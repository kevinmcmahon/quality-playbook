# BUG-013 — detect_skill_version uses substring match without line-start anchor

## Requirement
REQ-003 (Version parsers reject substring-only matches). Version-parsing helpers must
reject lines that only contain the substring `version:` (e.g., inside a quoted description
field). The match must be anchored at the start of a stripped line.
Doc source: `[source]` — EXPLORATION.md §F-1, Spec-Structured Parsing deep dive.

## Candidate origin
Source stage: new — Pass 2 requirement verification (REQ-003).

## Location
- File: `.github/skills/quality_gate/quality_gate.py`
- Lines: 182–191
- Function(s): `detect_skill_version`

## Symptom
When SKILL.md has a YAML frontmatter key `description: "The version: 1.4.5 release"` that
appears before the real `version: 9.9.9` line, `detect_skill_version` returns the value
extracted from the description field, not the version field. The gate validates the wrong
version string.

## Root cause
`detect_skill_version` uses `if "version:" in line` — a substring match with no anchoring.
The substring matches `description: "The version: 1.4.5 release"` because the literal
string `"version:"` appears within the description value. The function then applies
`re.sub(r".*version:\s*", "", line)` which strips everything up to and including the last
`version:` occurrence, returning the description text after it.
