# BUG-004 — archive_previous_run is non-atomic; control_prompts/ deleted not archived

## Requirement
REQ-009 (archive_previous_run is atomic and preserves control_prompts). The archive
operation must be crash-safe and must preserve `control_prompts/` alongside `quality/`.
Doc source: `[source]` — EXPLORATION.md §F-4, §QR-4.

## Candidate origin
Source stage: Phase 1 exploration / Pass 1 structural review.
Candidate bug ID: CB-4.

## Location
- File: `bin/run_playbook.py`
- Lines: 565–576
- Function(s): `archive_previous_run`

## Symptom
(1) If the system crashes between `copytree(quality_dir, archive_dir)` and `rmtree(quality_dir)`,
both the original `quality/` and the archive `previous_runs/TIMESTAMP/quality/` exist simultaneously.
On next start the runner may be confused about which is authoritative. (2) `control_prompts/` is
deleted (line 576) rather than archived alongside `quality/`. Diagnostic data from prior runs
(worker output, phase prompts) is permanently lost, impeding post-mortem debugging.

## Root cause
The archive sequence is: `copytree` → `rmtree(quality)` → `rmtree(control_prompts)`. There is
no atomic rename step; POSIX `rename(2)` would be atomic but the current code does not use it.
Line 576 calls `rmtree(control_prompts_dir)` with no prior copy — deletion is intentional but
incorrect per SKILL.md's archival contract.
