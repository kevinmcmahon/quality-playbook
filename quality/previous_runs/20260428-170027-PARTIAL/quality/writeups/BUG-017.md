# BUG-017 — Orchestrator prompts omit repo-root SKILL.md from source-checkout bootstrap

## 1. Summary

Both shipped orchestrator prompts omit plain repo-root `SKILL.md` from their setup search
lists even though the README tells maintainers to launch those prompts from the root checkout
of this repository. A maintainer following the documented bootstrap path can therefore fail
before Phase 1 starts.

## 2. Spec reference

REQ-018 — Orchestrator prompts honor source-checkout skill discovery.

> `[Tier 2] [README.md] §Agent launch` — "copy and paste the contents of ... quality-playbook.agent.md"

## 3. The code

`agents/quality-playbook.agent.md:35-43` and `agents/quality-playbook-claude.agent.md:45-55`
list installed-skill paths only. Neither list checks plain `SKILL.md` at the repo root, which
is the source-checkout layout the README points users toward.

## 4. Observable consequence

A maintainer who clones `quality-playbook`, opens the repo root, and pastes one of the shipped
orchestrator prompts can be told the skill is "not installed" even though the canonical
`SKILL.md` is right there in the checkout. The failure happens before any phase work starts, so
it looks like the bootstrap path itself is broken.

## 6. The fix

## 7. The test

`quality/test_regression.py::GapIterationRegressions::test_reg_gap17_agents_support_repo_root_skill`
asserts that both prompt files contain repo-root `SKILL.md`. On the current tree the test is an
expected failure because both files omit that entry. After the fix, removing the expected-failure
guard should make the test pass.
