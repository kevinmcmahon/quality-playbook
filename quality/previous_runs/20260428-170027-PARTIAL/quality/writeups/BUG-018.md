# BUG-018 — General orchestrator contradicts its own phase-execution ownership

## 1. Summary

The general orchestrator prompt says it must never execute phase logic itself, then later tells
Mode 1 to run Phase 1 "in the current session." Those two instructions cannot both be true, so
the prompt undermines the clean-context model it is supposed to enforce.

## 2. Spec reference

REQ-019 — General orchestrator phase ownership is internally consistent.

> `[Tier 2] [agents/quality-playbook.agent.md] §Role` — "You do NOT execute phase logic yourself"

## 3. The code

`agents/quality-playbook.agent.md:11-14` defines the role as spawn / verify / report only. But
`agents/quality-playbook.agent.md:77-81` says "Run Phase 1 in the current session." The role and
the mode instructions disagree about who owns Phase 1 execution.

## 4. Observable consequence

A tool using this prompt can collapse the phase boundary that the playbook depends on for depth:
the orchestrator becomes the phase executor instead of delegating or starting a fresh context.
The result is subtle quality degradation rather than an obvious crash.

## 6. The fix

```diff
--- a/agents/quality-playbook.agent.md
+++ b/agents/quality-playbook.agent.md
@@
-Run Phase 1 in the current session. When it completes, show the end-of-phase summary and tell the user to say "keep going" or "run phase N" to continue. Each subsequent phase should run in a **new session or context window** so it gets maximum depth.
+Start a fresh session or context for Phase 1. When it completes, show the end-of-phase summary and tell the user to say "keep going" or "run phase N" to continue. Each subsequent phase should also run in a **new session or context window** so it gets maximum depth.
```

## 7. The test

`quality/test_regression.py::GapIterationRegressions::test_reg_gap18_general_agent_keeps_context_ownership_consistent`
asserts that the prompt does not simultaneously contain both contradictory instructions. It
fails on the current prompt and becomes green once one ownership model remains.
