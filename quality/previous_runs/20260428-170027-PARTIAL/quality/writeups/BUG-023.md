# BUG-023 — quality_gate.py misclassifies the bootstrap repo because of fixture repos

## 1. Summary

The gate’s language heuristic scans nested benchmark fixture repos under `repos/`, so the
bootstrap self-audit can be classified as Go before the validator even looks at the root Python
runner and gate code.

## 2. Spec reference

REQ-024 — quality_gate.py detects the audited project language, not fixture languages.

> `[Tier 2] [source] quality/results/quality-gate.log` — "test_functional.py does not match project language (go)"

## 3. The code

`.github/skills/quality_gate/quality_gate.py:209-250` excludes only `vendor`, `node_modules`,
`.git`, and `quality` while walking the tree for language detection.

## 4. Observable consequence

The live bootstrap repo currently fails `check_test_file_extension()` even though its functional
tests are correctly written in Python. That false FAIL hides real gate regressions behind fixture
noise.

## 6. The fix

```diff
--- a/.github/skills/quality_gate/quality_gate.py
+++ b/.github/skills/quality_gate/quality_gate.py
@@ -223,7 +223,7 @@ def detect_project_language(repo_dir):
         ("c", ".c"),
         ("agc", ".agc"),
     ]
-    excluded = {"vendor", "node_modules", ".git", "quality"}
+    excluded = {"vendor", "node_modules", ".git", "quality", "repos"}
```

## 7. The test

`quality/test_regression.py::UnfilteredIterationRegressions::test_reg_unfiltered23_language_detection_ignores_fixture_repos`
