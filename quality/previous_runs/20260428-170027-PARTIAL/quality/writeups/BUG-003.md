# BUG-003 — Phase 2 gate uses threshold 80 WARN instead of 120 FAIL

## Requirement
REQ-004 (Phase 1 line-count gate enforces 120 as FAIL). `check_phase_gate("2")` must
return `ok=False` when EXPLORATION.md has fewer than 120 lines. The failure message must
say `GATE FAIL`, not `GATE WARN`.
Doc source: `[SKILL.md §Phase 1 completion gate, lines 906,916]`.

## Candidate origin
Source stage: Phase 1 exploration / Pass 1 structural review.
Candidate bug ID: CB-3.

## Location
- File: `bin/run_playbook.py`
- Lines: 455–457
- Function(s): `check_phase_gate`

## Symptom
An EXPLORATION.md with 85 lines passes the Phase 2 entry gate without stopping the run.
The runner prints a `GATE WARN` message but sets `ok=True`, allowing Phase 2 to proceed
with a thin exploration. SKILL.md's self-gate check at line 916 independently requires 120
lines and would FAIL the run — creating an inconsistency between the runner gate and the
skill's embedded self-gate.

## Root cause
The threshold at `run_playbook.py:455` is `80` instead of `120`. The gate emits a WARN
message but returns `ok=True` — the gate is non-blocking. SKILL.md:906 specifies 120 as
the minimum for substantive exploration content; the runner was not updated when this
threshold was documented.
