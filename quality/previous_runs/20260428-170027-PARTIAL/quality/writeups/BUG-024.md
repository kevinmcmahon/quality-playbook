# BUG-024 — File-existence gate omits valid functional-test filenames

## 1. Summary

The quality gate can falsely report that a repo is missing its functional-test artifact when that
repo uses the documented Go filename `quality/functional_test.go`.

## 2. Spec reference

REQ-025 — File-existence gate recognizes every documented functional-test filename.

> `[Tier 1] [functional_tests.md] §Writing Functional Tests` — "functional_test.go (Go)"

## 3. The code

`.github/skills/quality_gate/quality_gate.py:299-303` hardcodes a smaller filename set than the
published functional-test contract, omitting `functional_test.*` even though `functional_tests.md`
names `functional_test.go` as the Go convention.

## 4. Observable consequence

A valid Go playbook run can FAIL the gate before any deeper validation occurs, even though the
helper library and the docs both agree that `functional_test.go` is the correct artifact name.

## 6. The fix

```diff
--- a/.github/skills/quality_gate/quality_gate.py
+++ b/.github/skills/quality_gate/quality_gate.py
@@
-    if has_file_matching(q, ["test_functional.*", "FunctionalSpec.*",
-                             "FunctionalTest.*", "functional.test.*"]):
+    if has_file_matching(q, ["test_functional.*", "functional_test.*",
+                             "FunctionalSpec.*", "FunctionalTest.*",
+                             "functional.test.*"]):
         pass_("functional test file exists")
     else:
-        fail("functional test file missing (test_functional.*, FunctionalSpec.*, FunctionalTest.*, functional.test.*)")
+        fail("functional test file missing (test_functional.*, functional_test.*, FunctionalSpec.*, FunctionalTest.*, functional.test.*)")
```

## 7. The test

`quality/test_regression.py::ParityIterationRegressions::test_reg_parity24_file_existence_accepts_functional_test_go`
creates a minimal repo whose only functional test is `quality/functional_test.go` and asserts
that `check_file_existence()` leaves the FAIL counter at zero.
