# BUG-014 — validate_iso_date rejects valid ISO 8601 datetimes

## Requirement
REQ-007 (ISO 8601 validation matches field grammar). `validate_iso_date` must accept full
ISO 8601 datetime strings like `2026-04-18T23:43:14Z` in addition to date-only strings.
Run-metadata `start_time` / `end_time` fields use the datetime form.
Doc source: `[SKILL.md §Sidecar JSON Canonical Examples, line 154]`; `[SKILL.md §Run Metadata, line 187]`.

## Candidate origin
Source stage: new — Pass 2 requirement verification (REQ-007).

## Location
- File: `.github/skills/quality_gate/quality_gate.py`
- Lines: 156–173
- Function(s): `validate_iso_date`

## Symptom
The quality gate reports `start_time` as `bad_format` for a run whose metadata JSON
contains `"start_time": "2026-04-18T23:43:14Z"` — a valid ISO 8601 datetime. The gate
FAILs on valid metadata, creating false negatives that block CI on correct runs.

## Root cause
`validate_iso_date` uses `re.fullmatch(r"\d{4}-\d{2}-\d{2}", date_str)`. `fullmatch`
requires the entire string to match the pattern. A datetime string like
`2026-04-18T23:43:14Z` contains 20 characters, while the pattern matches only 10
(`YYYY-MM-DD`). The fullmatch fails and the function returns `"bad_format"`.
