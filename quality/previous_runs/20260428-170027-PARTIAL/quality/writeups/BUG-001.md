# BUG-001 — skill_version() rejects bold Version form

## Requirement
REQ-001 (Single canonical version-parser helper). `benchmark_lib.skill_version()` must
handle both the bare `version: X.Y.Z` and bold `**Version:** X.Y.Z` SKILL.md forms.
Doc source: `[source]` — EXPLORATION.md §F-1, §QR-1.

## Candidate origin
Source stage: Phase 1 exploration / Pass 1 structural review.
Candidate bug ID: CB-1.

## Location
- File: `bin/benchmark_lib.py`
- Lines: 100–116
- Function(s): `skill_version`

## Symptom
When SKILL.md uses the bold frontmatter form `**Version:** 1.4.5`, `skill_version()`
returns `None`. The bare-name version-append fallback in `resolve_target_dirs`
(`run_playbook.py:217`) then silently skips the version-append, and the user sees a
cryptic "is not a directory" error instead of the expected augmented path resolution.

## Root cause
`skill_version` iterates lines and checks `stripped.startswith("version:")` (case-sensitive).
The bold form `**Version:** 1.4.5` does not start with `version:` — it starts with `**`.
The function exits the loop and returns `None`. The shared `VERSION_PATTERN` regex at
line 37 handles both forms correctly via `(?:version:|\*\*Version:\*\*)` but `skill_version`
does not use it.
