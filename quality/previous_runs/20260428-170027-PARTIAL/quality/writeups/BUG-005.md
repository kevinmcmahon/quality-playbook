# BUG-005 — PROTECTED_PREFIXES missing AGENTS.md

## Requirement
REQ-006 (AGENTS.md is protected from silent cleanup reversion). `cleanup_repo` must not
silently revert edits to `AGENTS.md`. Either `AGENTS.md` is added to the protected set or
the cleanup function gains a mechanism to refuse reversion of required artifacts.
Doc source: `[SKILL.md §Complete Artifact Contract, lines 106,122]`.

## Candidate origin
Source stage: Phase 1 exploration / Pass 1 structural review.
Candidate bug ID: CB-5.

## Location
- File: `bin/benchmark_lib.py`
- Lines: 177–182
- Function(s): `PROTECTED_PREFIXES` constant, `_is_protected`

## Symptom
An AI agent that writes or updates `AGENTS.md` during Phase 2 has that change silently
reverted by `cleanup_repo` at the end of the session. The user sees no warning. The next
`quality_gate.py` run then reports `AGENTS.md missing` as a FAIL, even though the agent
produced it.

## Root cause
`PROTECTED_PREFIXES` uses directory-prefix matching. `AGENTS.md` at the project root does
not start with any of the four protected prefixes (`quality/`, `control_prompts/`,
`previous_runs/`, `docs_gathered/`). The `_is_protected("AGENTS.md")` call returns False,
allowing `cleanup_repo` to revert it via `git checkout`.

## Proposed fix

```diff
--- a/bin/benchmark_lib.py
+++ b/bin/benchmark_lib.py
@@ -177,7 +177,9 @@ PROTECTED_PREFIXES = (
     "quality/",
     "control_prompts/",
     "previous_runs/",
     "docs_gathered/",
 )
+PROTECTED_EXACT = ("AGENTS.md",)

-    return any(path.startswith(prefix) for prefix in PROTECTED_PREFIXES)

```
