# BUG-007 — docs_present accepts noise files (.DS_Store, empty) as documentation

## Requirement
REQ-011 (docs_present requires substantive documentation). `docs_present(repo_dir)` must
verify at least one non-hidden, non-empty documentation file in `docs_gathered/`.
Doc source: `[source]` — EXPLORATION.md §F-6.

## Candidate origin
Source stage: Phase 1 exploration / Pass 1 structural review.
Candidate bug ID: CB-7.

## Location
- File: `bin/run_playbook.py`
- Lines: 560–562
- Function(s): `docs_present`

## Symptom
A `docs_gathered/` directory containing only `.DS_Store` (macOS filesystem metadata) causes
`docs_present` to return True. The runner proceeds as if documentation is present; the agent
begins a documentation-grounded exploration when no actual documents exist.

## Root cause
`docs_present` uses `any(docs_dir.iterdir())`. The `iterdir()` generator yields `.DS_Store`
as a `Path` object. `any()` evaluates the Path as truthy (non-None) and returns True
immediately. There is no check for hidden-file prefix, file extension, or non-zero size.

+++ b/bin/run_playbook.py

     docs_dir = repo_dir / "docs_gathered"
