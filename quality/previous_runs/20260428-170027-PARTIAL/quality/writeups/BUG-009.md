# BUG-009 — _pkill_fallback missing gh copilot -p pattern

## Requirement
REQ-012 (Kill-path covers gh copilot workers). `_pkill_fallback` must include a pattern
for `gh copilot -p` workers. When the parent crashes during a `--copilot` benchmark,
`--kill` must terminate all orphaned copilot workers.
Doc source: `[source]` — EXPLORATION.md §F-9, §QR-5.

## Candidate origin
Source stage: Phase 1 exploration / Pass 1 structural review.
Candidate bug ID: CB-9.

## Location
- File: `bin/run_playbook.py`
- Lines: 808–829
- Function(s): `_pkill_fallback`

## Symptom
When the parent process crashes during a `--copilot` benchmark run, orphaned
`gh copilot -p <prompt> --yolo` processes continue executing on the machine. Running
`python3 bin/run_playbook.py --kill` does not terminate them because `_pkill_fallback`
has no pattern matching `gh copilot`.

## Root cause
The patterns list at `run_playbook.py:808-816` has three entries:
`["bin/run_playbook.py", "claude -p", "claude --model"]`. The `gh copilot` command is
dispatched by `command_for_runner("copilot", ...)` and produces a process matching
`gh copilot --yolo --model <model> -p <prompt>`. No entry in the patterns list matches
the `gh` binary or the `copilot` subcommand.
