# BUG-016 — Phase 5 entry gate does not enforce SKILL.md Phase 4 completion requirements

## 1. Summary

`check_phase_gate("5")` in `bin/run_playbook.py:473-478` fails to enforce the mandatory
hard-stop conditions that SKILL.md:1663-1667 requires before Phase 5 may proceed. A run
with an empty `spec_audits/` directory (no triage file, no auditor files) is allowed to
proceed to Phase 5 with `ok=True` and zero warnings, provided `PROGRESS.md` exists.

## 2. File and line

`bin/run_playbook.py:473-478`

## 3. Requirements

REQ-010 — Phase 3/4/5 entry gates require all Phase 2 / prior-phase artifacts.

> "Phase 5 gate checks all prior plus whatever Phase 4 produces."

SKILL.md:1663-1667 — Phase 5 entry gate (mandatory — HARD STOP):

> 1. `quality/spec_audits/` directory exists and contains at least one `*triage*` file
> 2. `quality/spec_audits/` contains at least one `*auditor*` file
> 3. `quality/PROGRESS.md` exists and its Phase 4 line is marked `[x]`
>
> "If any of these are missing, STOP and go back to Phase 4."

## 4. Reproduction

```python
import tempfile, pathlib

with tempfile.TemporaryDirectory() as td:
    td = pathlib.Path(td)
    q = td / "quality"
    q.mkdir()
    (q / "PROGRESS.md").write_text("- [x] Phase 1\n- [x] Phase 2\n- [x] Phase 3\n- [ ] Phase 4\n")
    (q / "BUGS.md").write_text("# Bugs\n### BUG-001 — some bug\n")
    (q / "spec_audits").mkdir()  # exists but NO triage file, NO auditor files
    # Per SKILL.md:1663-1667 this should return ok=False
    # Current code: returns GateCheck(ok=True, messages=[])
    result = check_phase_gate(q, "5")
    print(result)  # GateCheck(ok=True, messages=[]) — incorrect
```

## 5. Root cause

The Phase 5 branch at `run_playbook.py:476` checks:
```python
if not (quality_dir / "BUGS.md").is_file() and not (quality_dir / "spec_audits").is_dir():
    messages.append("GATE WARN Phase 5: ...")
```

This compound condition uses `and` — both must be absent to trigger the warning. If
`spec_audits/` is a directory (even empty), the right side is False and no warning fires.
The function returns `GateCheck(ok=True, messages=[])`. The Phase 4 `[x]` checkbox is
never inspected.
