# BUG-020 — Missing docs block code-only runs

## 1. Summary

The runner treats missing `docs_gathered/` as a hard skip in both phase mode and single-pass mode.
That blocks the documented code-only fallback path entirely.

## 2. Spec reference

REQ-021 — Missing gathered docs warn and continue with code-only analysis.

> `[Tier 1] [SKILL.md] §Documentation warning` — "I'll proceed with code-only analysis"

## 3. The code

`bin/run_playbook.py:645-649` and `671-675` both print `SKIP: ... docs_gathered/ is missing or empty`
and return `1` before any prompt is launched.

## 4. Observable consequence

A user auditing a code-only repo cannot start the playbook at all from the shipped runner. The
documented "warn and continue" behavior is unreachable in practice.
