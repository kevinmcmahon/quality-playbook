# BUG-025 — Extension checker only validates `test_functional.*`

## 1. Summary

The quality gate skips extension validation for valid language-native functional-test filenames
such as `FunctionalTest.java`, `FunctionalSpec.scala`, and `functional.test.ts`.

## 2. Spec reference

REQ-026 — Extension checker validates the discovered language-appropriate test file.

> `[Tier 1] [SKILL.md] §What This Skill Produces` — "FunctionalSpec.scala ... FunctionalTest.java"

## 3. The code

`.github/skills/quality_gate/quality_gate.py:795-839` calls `first_file_matching(q,
["test_functional.*"])` and warns `No test_functional.* found` if that one prefix is absent.

## 4. Observable consequence

Java, Scala, and TypeScript repos can pass the file-existence check yet still bypass extension
validation entirely, so the gate never verifies that their functional-test artifact matches the
detected project language.

## 6. The fix

```diff
--- a/.github/skills/quality_gate/quality_gate.py
+++ b/.github/skills/quality_gate/quality_gate.py
@@
-    func_test = first_file_matching(q, ["test_functional.*"])
-    reg_test = first_file_matching(q, ["test_regression.*"])
+    func_test = first_file_matching(q, ["test_functional.*", "functional_test.*",
+                                        "FunctionalSpec.*", "FunctionalTest.*",
+                                        "functional.test.*"])
+    reg_test = first_file_matching(q, ["test_regression.*"])
@@
-        warn("No test_functional.* found")
+        warn("No functional test file found across the supported naming matrix")
         return
@@
-        pass_(f"test_functional.{ext} matches project language ({detected_lang})")
+        pass_(f"{func_test.name} matches project language ({detected_lang})")
```

## 7. The test

`quality/test_regression.py::ParityIterationRegressions::test_reg_parity25_extension_check_accepts_functionaltest_java`
creates a minimal Java repo with `quality/FunctionalTest.java` and asserts that the checker emits
no warning and instead reports a language-match PASS for that discovered filename.

## 8. Related issues

The adversarial iteration surfaced helper-side filename drift too. BUG-026 removes the undocumented
functional alias `test_functional_test.*`, and BUG-027 keeps regression discovery canonical so the
gate's fix does not inherit helper-only aliases.
